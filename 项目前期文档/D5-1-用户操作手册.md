# GRC 平台 · 用户操作手册（按角色）

**文档编号：** D5-1 ｜ **版本：** v1.0（草案） ｜ **日期：** 2026-06-23
**适用：** 平台各角色用户 ｜ **状态：** 待评审

> 本手册按角色组织常用操作，配合系统界面使用。所有"查看"默认仅限本子公司范围（集团管理层/只读审阅角色除外）。

---

## 一、通用
- **登录**：用 AD 账号登录（或单点登录）。
- **导航**：左侧菜单按"概览 / 业务 / 系统管理"分组；点击进入对应模块。
- **主题**：右上角"主题"可切换界面风格（不影响数据）。
- **AI 助手**：左下角悬浮按钮，随时提问（回答按你的权限范围）。
- **通知**：右上角铃铛查看；企微/飞书/邮件按规则接收。

---

## 二、普通员工（制度签署方）
- **阅读制度**：制度体系 → 查看适用于你的制度。
- **签署确认**：收到提醒后，打开制度 → 阅读 → 点"已阅读并知悉确认"。⚠️ 逾期未确认会提醒你及上级。

## 三、系统/业务责任人
- **填写风险自评**：风险评估 → 我的评估 → 填写表单（按题作答，可上传佐证）→ 提交（进入审核）。
- **承接整改**：整改任务 → 处理 → 完成后上传证明 → 等发起人验证。⚠️ 未上传证明不可关闭。

## 四、部门级评估审核人
- **审核评估**：风险评估 → 待审核 → 查看 → 通过/退回。
- **确认整改验证**：整改任务 → 待验证 → 核对证明 → 验证通过/退回。

## 五、信息安全负责人（评估侧）
- **审批报告**：风险评估 → 报告审批 → 复核 → 签批生效。
- **配置打分模型/风险矩阵**：由模板配置入口设置（见管理员手册）。

## 六、审计执行人员
- **执行检查表**：内部审计 → 我的审计 → 填写检查表 → 记录发现（描述/依据条款/严重度/上传证据）。
- **整改跟踪**：发现项转整改（复用整改任务）。

## 七、审计计划制定人 / 报告审批人
- **制定计划**：内部审计 → 审计计划 → 新建 → 排期 → 关联对象与检查表。
- **签批报告**：审计 → 报告 → 复核签批。

## 八、合规分析人员 / 法规库维护人员
- **处理法规更新**：法规跟踪 → 更新流 → 查看变更摘要 → 判断对内部制度影响。
- **维护映射**：登记法规条款 ↔ 制度条款映射。

## 九、供应商管理人员
- **维护供应商**：第三方供应商 → 台账 → 新增/编辑；发起准入/续约/退出评估。
- **事件触发复评**：登记外部事件（如负面舆情）→ 发起复评。
- **SLA 跟踪**：查看 SLA 状态与到期提醒。

## 十、子公司/集团管理层
- **查看看板**：合规态势总览 → 按子公司/时间下钻查看风险、整改、审计态势。⚠️ 子公司管理层仅本子公司；集团管理层可看全集团。

## 十一、AI 智能问答（全角色）
- 提问示例："对比 A、B 子公司近两次评估的高风险项变化"。
- 回答会标注数据来源；超出你权限范围的数据不会返回。

---

## 十二、常见问题
| 问题 | 说明 |
|---|---|
| 看不到某子公司数据 | 受组织权限限制，属正常 |
| 表单提交不了 | 检查必填项；网络/会话；草稿会自动保存 |
| 没收到通知 | 确认通知偏好与通道；联系管理员核对规则 |
