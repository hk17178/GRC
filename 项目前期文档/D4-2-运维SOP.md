# GRC 平台 · 运维 SOP（标准作业程序）

**文档编号：** D4-2 ｜ **版本：** v1.9（草案） ｜ **日期：** 2026-06-23
**目标：** 每个动作"执行某脚本 → 看到某输出 → 完成"，**小白照做即可**
**状态：** 待评审

> ⌨️ 命令 ｜ ✅ 预期 ｜ ⚠️ 注意。所有操作在部署目录 `mandao-grc/` 下执行。

---

## 一、日常巡检 SOP（每日）
```bash
./doctor.sh             # ⌨️ 一键体检
```
✅ 输出各项为 `[OK]`：容器状态、磁盘水位、数据库连通、Redis、MinIO、各身份源（域）连通与同步状态（逐域）、短信/企微等外部连通、爬虫健康、AI 模型端点/密钥/RAG/数据不出域开关、operation_log 完整性、代码仓库同步连通、定时任务健康、最近备份时间、队列积压。
⚠️ 出现 `[WARN]/[FAIL]` 时对照 §七 故障速查处置。

巡检要点（doctor 已覆盖，人工复核）：
- [ ] 所有容器 Up/healthy
- [ ] 磁盘使用 < 80%
- [ ] 最近一次备份成功且在 24h 内
- [ ] 通知发送成功率正常、无大量失败
- [ ] 爬虫各源健康（无连续抓取失败）
- [ ] **各身份源（域）逐域连通与同步状态正常**（每个 identity_source 单独核验；某域中断不影响其它域）
- [ ] **operation_log 完整性/不可篡改性核验通过**（哈希链/链式校验无断点）
- [ ] **模型接入健康**：模型端点连通、API Key 有效期（临期预警）、RAG 索引状态、"数据不出域"开关状态（默认开，核对未被关闭）
- [ ] **建议与反馈（M-FB）链路健康**：代码仓库（GitHub/内网 Git）同步连通、反馈周期分析推送定时任务健康

## 二、备份 SOP（每日自动 + 按需手动）
```bash
./backup.sh             # ⌨️ 全量备份（数据库 + MinIO + 配置）
```
✅ 生成 `backups/grc-YYYYMMDD-HHMM.tar.gz`，末尾打印 `Backup OK`。
- 自动备份：默认每日凌晨执行（cron/计划任务，安装时配置）。
- ⚠️ **证据留痕数据为监管举证核心，不可丢**（需求 14.2）：备份须异地/异盘留存一份。
- **配置一并全量备份**：高自由度可配置内容（CR-001-2）——自定义字段元数据登记、自定义视图/筛选、看板、报表、工作流、KPI、通知场景库——以及 模型接入与管理配置（端点/RAG/提示词/开关，密钥不落明文、仅引用密钥库），均与业务库（含 operation_log）在同一备份包内全量备份，确保恢复后配置与业务数据一致。
- 保留策略：日备保留 N 天、月备长期（按监管留存期，见 §六）。

## 三、恢复 SOP
```bash
./restore.sh backups/grc-YYYYMMDD-HHMM.tar.gz    # ⌨️
```
✅ 提示 `Restore OK`，服务自动重启并健康检查通过。
⚠️ 恢复为**覆盖式**，执行前确认目标环境与备份版本匹配；生产恢复前先在预生产演练。
- RTO/RPO：目标值在设计阶段确认；演练频率建议每季度一次。

---

## 四、升级 SOP
```bash
# 离线环境：先导入新版镜像
docker load -i mandao-grc-offline-<新版本>.tar    # ⌨️（离线）
./upgrade.sh                                       # ⌨️
```
✅ 流程自动执行：备份 → 拉/切新镜像 → 数据库迁移（Flyway，向前兼容）→ 滚动重启 → 健康校验 → 打印 `Upgrade OK <版本>`。
⚠️ 升级会自动备份；升级中进行中的评估不受影响（模板版本快照 + DB 兼容迁移）。

### 回滚 SOP
```bash
./rollback.sh <上一版本>    # ⌨️
```
✅ 回滚镜像 + 数据库回滚点，健康校验通过。
⚠️ 仅在升级后出现严重问题时执行；回滚前保留现场日志（§五）。

## 五、日志收集 SOP
```bash
./logs.sh [服务名] [时间范围]    # ⌨️ 例：./logs.sh grc-app 1h
```
✅ 导出结构化日志到 `logs/export/`，便于排障或上报。
- 审计日志（operation_log）独立、不可篡改，**不随普通日志清理**（D1-3 §8）。
- 日志可对接 ELK/Loki（可选）。

## 六、容量与归档 SOP（每月）
- 检查磁盘与 MinIO 用量；附件/证据增长快时扩容（加盘/迁移 volume）。
- **留痕到期处理**：到期数据**人工确认后归档**，不自动清除（需求 11.4，避免监管延期检查无法提供）。
- 月度：核对备份完整性、抽查一次恢复演练（季度）。

## 七、故障速查与处置（Playbook）
| 现象 | 处置 |
|---|---|
| 某容器异常退出 | `docker compose restart <服务>`；看 `logs.sh`；自愈策略一般会自动重启 |
| 数据库连接失败 | 检查 postgres 容器/磁盘；连接数；必要时重启 |
| 页面 502/打不开 | 检查 nginx 与 grc-app 健康；端口放通 |
| 通知大量失败 | 核对通道凭证/额度；企微/飞书是否被限流（见限流告警）；短信额度 |
| **爬虫失效告警** | 目标网站改版可能：更新该源采集规则配置（无需改码）；连不通则查网络/反爬（需求 6.3） |
| KRI 数据缺失 | 检查 SIEM/日志/漏洞接口连通与采集任务（GRC 仅接口采集 KRI，漏洞闭环/SIEM 本体不在本平台） |
| 磁盘将满 | 清理过期普通日志（勿动审计/留痕）；扩容；归档 |
| **某身份源（域）同步中断** | 仅影响该域，**权限快照维持中**（不依赖域间信任，其它域不受影响）；查该 identity_source 的 AD/LDAP 域控连通、绑定账户与凭证；恢复后触发该域增量同步；勿因单域故障重启全局 |
| AI 服务异常（端点） | 核心业务不受影响（降级）；查 grc-ai 容器、模型端点连通与出站白名单（防 SSRF）是否放通 |
| AI 服务异常（密钥） | 核对 API Key 是否过期/被吊销；密钥从密钥库读取（不落明文）；在 模型接入与管理中更新后重试 |
| AI 服务异常（RAG） | 查 RAG 索引是否损坏/未构建；按需重建索引；确认"数据不出域"开关状态符合预期（默认开） |
| 反馈同步失败（M-FB） | 查代码仓库（GitHub/内网 Git）同步连通与凭证（凭证入密钥库）；出站受管理员审批+脱敏+出站审计约束；必要时一键切内网 Git；查反馈周期分析推送定时任务是否健康 |
| 越权/安全告警 | 立即查 operation_log（先核验其完整性/不可篡改性），按 D4-3 应急预案处置 |

## 八、监控与告警（复用 M10）
关键指标阈值告警，经通知中心送达运维：
- 可用性/容器健康、队列积压、爬虫连续失败、通知失败率、留痕写入异常、磁盘水位、备份失败、越权拦截激增。
- **某身份源同步中断（权限快照维持中）**：逐域监控，单域同步失败即告警，明确标注该域降级范围与"权限快照维持中"，不影响其它域。
- **operation_log 完整性告警**：完整性/不可篡改性核验失败（链式哈希断点/被改）即高优先级告警。
- **模型接入告警**：模型端点不可达、API Key 临期/失效、RAG 索引异常、"数据不出域"开关被关闭（默认应为开）。
- **建议与反馈（M-FB）告警**：代码仓库同步连通中断、反馈周期分析推送定时任务失败/未按时执行。

## 九、集群形态运维（生产，可选）
- 扩容应用：增加 `grc-app` 副本，Nginx 自动纳入负载。
- 数据库：主备状态巡检、故障转移演练。
- 滚动升级：逐副本灰度，保证不中断。
- ⚠️ 集群操作需具备容器编排基础，建议由 DevOps 执行。
