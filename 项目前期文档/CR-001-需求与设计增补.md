## CR-001-1 新增「外部审计」模块（M3 扩展）⭐
**决策：** 平台不仅管内部审计，还须管理**外部审计 / 监管检查 / 认证审计**的闭环。将 M3 升级为统一「审计管理」，以 `审计类型` 区分：
- 内部审计 / **外部审计（ISO、PCI、ISO27701 等认证机构审计）** / **监管检查（PBOC、网信办等）** / 认证审计。

**外部审计闭环范围：**
1. 登记外审任务（认证体系、外部审计机构、责任单位、周期）；
2. 录入**外部机构的审计发现**（来源外审、问题、依据、严重度、责任单位）；
3. 针对发现做**整改计划管理 + 整改任务分配**（复用 M2 工单机制）；
4. **整改状态经企业微信（M10）通知责任人**，期限临近/逾期自动提醒并可升级上级；
5. 向监管/认证机构**回复 → 闭环**；认证有效期跟踪与到期提醒。

**权限隔离（关键）：** **各子公司管理本子公司各认证体系的外部审计，集团管理集团级认证；数据按组织严格隔离、权限分开**（沿用 M8 四元组 + 组织隔离，D1-3 §5）。

**影响文档：** 需求 §五（M3 扩为内/外审）；D1-2（audit 表增 `audit_type`、外部机构、外部发现）；D1-4（M3 章节补外审闭环）；D1-7（新增「外部审计」页规格）；DM-1 RTM。
**原型：** 已新增「外部审计闭环管理」模块（外审任务 / 外部发现 / 整改跟踪 + 企微通知状态）。

## CR-001-2 「高自由度 / 反硬编码」升格为第一性产品原则 ⭐
**决策：** 用户核心诉求——**尽量多的自由度、更精细化地管理与自定义各模块**。将"配置化、反硬编码"由架构策略**升格为显式产品第一性原则**，并扩展可配置面：

| 已有可配置 | 新增可配置面（扩展） |
|---|---|
| 表单模板/打分/报告映射、通知规则、角色、体系、法规源 | **自定义字段**（在对象上加业务字段）、**自定义列表视图与筛选**、**自定义看板**、**自定义报表模板**、**自定义工作流/审批流**、**自定义 KPI/指标**、**自定义通知场景与文案** |

**配套约束（防过度自由致混乱）：** 沿用表单引擎"**集中管控、分级开放**"治理思路全局化——结构性/底线项归口管理员 + 发布前预览测试；自定义项纳入 M8 权限与变更留痕。
**数据层预留：** 自定义字段需在数据模型预留扩展机制（如 JSONB 扩展列 + 字段元数据登记），**必须在数据模型阶段考虑，后期补救成本高**。

**影响文档：** 需求 §13.5 升格表述；D1-1（新增"高自由度可配置"为质量目标 Q）；D1-2（扩展字段元数据、自定义视图/看板存储）；D1-6 思路全局化；新增可考虑 D1-8「可配置性/低代码专项设计」。
**已落地示例（原型）：** 「合规态势」大屏支持**自定义添加组件**（指标卡/图表/信息流 组件库，可自选信息流上屏）——即"自定义看板"的具体体现；后续把"自定义字段/视图/报表/工作流/KPI"逐项纳入。

## CR-001-3 国际化 / 中英文切换（i18n）⭐缺口
**决策：** 平台适用范围含**海外业务子公司**，需支持**中 / 英文切换**。
**落地要点：** 前端文案**外置语言包**（不硬编码文案）、日期/数字/时区**本地化**、制度等内容支持**可选双语**；顶栏提供语言切换。
**影响文档：** 需求新增「国际化」非功能需求；D1-1（i18n 架构：文案资源化、locale 上下文）；D1-7（顶栏语言切换控件规格）。
**原型：** 待补语言切换按钮（下一步）。

## CR-001-4 通知内容场景库（企业微信为主）
**决策：** 接收人涉及各种角色与管理层级，通知**内容与场景丰富且难以一次穷尽**，故按**可配置场景库**实现（呼应 CR-001-2）：场景 → 触发条件 → **接收角色/层级**（精细路由）→ **可编辑消息模板（含变量、跳转）**，新增场景无需改码。
**代表场景：** 整改期限临近→责任人；整改逾期→责任人+上级（**升级链**）；内/外审或评估**发现概况**→子公司负责人；**法规变化**→该子公司合规对接人；KRI 预警；制度待签署；**管理层合规简报（聚合，不逐条打扰）**。
**通道：** 通道仍为需求锁定的 4 类（企微/飞书/邮件/短信，单向）；本项丰富的是**内容与触达精细度**，并补**站内待办中心**作为站内触达。
**影响文档：** 需求 §12 通知内容补"场景库 + 升级链 + 管理层摘要"；D1-4 M10；D1-7 通知场景页。
**原型：** 已新增「企微通知场景库」(场景表 + 消息模板预览 + 可配置说明)。

## CR-001-5 业务菜单顺序调整
**决策：** 业务分组由上到下：**外部审计 → 内部审计 → 风险评估 → 法规跟踪 → 制度体系 → AI 智能问答 → 第三方供应商**（制度体系置于法规跟踪与 AI 之间，已确认）。
**附带 UI 文案决策：** ① 顶部页签/标题「合规态势总览」简化为「**合规态势**」；② 系统界面**不出现实现/设计术语类说明文字**（如"两层解耦""四元组""复用工单机制""M10/Flowable""→闭环"流程叙述、组织隔离说明段等），此类属设计文档语言，不进产品 UI。原型已据此清理各模块的说明性 infoline。
**影响文档：** D1-7 导航信息架构；原型已调整。

## CR-001-6 法规追踪源「自动分类排版」
**决策：** 新增追踪源后，采集内容**自动按体系/主题分类**并**结构化排版**到统一卡片流（结合 AI 分类与条款级摘要），降低人工整理成本。
**影响文档：** 需求 §六 M4 补"自动分类与结构化排版"；D1-4 M4；D1-5（爬虫采集后处理：AI 分类/摘要）。
**原型：** 法规页已加"新增追踪源（自动分类排版）"入口说明。

## CR-001-7 风险等级改为五级 ⭐
**决策：** 风险评估等级由原 3 级（高/中/低）统一改为**五级（由低到高）：极低、低、中、高、极高**。
> 说明：原 3 级为实现选择，需求未硬性固定级数；现按业务要求固化为五级。
**落地要点：** 更新打分模型 `levelMatrix`（风险值→五级区间）、数据模型 `risk_level` 枚举、看板/标签/分布图的等级口径；M9 聚合口径同步统一（避免子公司口径不一致致汇总失真，需求 11.4）。
**影响文档：** D1-2（risk_level 枚举，已更新）；D1-6（scoring_model 五级矩阵）；D1-7（标签/分布密度）。
**原型：** 风险标签与分布图已改五级。
