# GRC 平台 · 环境与配置管理

**文档编号：** D2-3 ｜ **版本：** v1.0（草案） ｜ **日期：** 2026-06-23
**依据：** D1-1 架构、D4-1 安装 ｜ **状态：** 待评审

## 一、环境规划
| 环境 | 用途 | 数据 | 外部系统 |
|---|---|---|---|
| 开发 | 本地/共享开发 | 造数 | mock/stub |
| 测试 | 功能/集成/系统测试 | 脱敏测试数据 | mock + 部分真连 |
| 预生产 | 性能/安全/UAT/演练 | 接近生产（脱敏） | 真连测试端点 |
| 生产 | 正式运行 | 真实 | 真连 |

- 四环境**容器化一致**，差异仅在配置（连接、参数、规模）。

## 二、配置管理
- 配置分层：镜像内默认 + 环境配置（`config/`）+ 运行时（Web 向导/管理后台）。
- **敏感配置**（DB 密码、各 identity_source 独立服务账号与密钥（按域隔离存密钥库）、短信密钥、模型 API 端点/密钥、代码仓库同步凭证）经密钥库/加密注入，**不明文、不入库 Git**（D1-3 §10）。
- 配置版本化，变更可回退（D4-3 §四）。

## 三、CI/CD
- 流水线见 D2-2 §七；制品=容器镜像，跨环境同一镜像、换配置。
- 部署：测试/预生产自动化；生产按发布流程（D4-1 升级 SOP）。

## 四、环境就绪检查
各环境用 `doctor.sh` 自检；外部依赖按 D2-1 §六锁定后接入。
- **多域控逐域连通校验**：`doctor.sh` 增逐 identity_source 连通性探测（LDAP(S) 多控制器可达、Base DN 可绑定、服务账号密钥可解析），单域故障仅告警该域、不阻断其它域。

## 五·补（V1.9 对齐 · CR-003）多域控连接配置

平台对接 N 个 `identity_source`（各子公司 / 海外独立 AD 域 / 林），各域配置独立、按域隔离。每个 identity_source 配置项：

| 配置项 | 说明 |
|---|---|
| 域/林（domain/forest） | 域标识 `domain_id`，林归属；登录路由按 `DOMAIN\\user` 或 `user@domain` 匹配 |
| LDAP(S) 多控制器 | 主备多个域控制器地址（优先 LDAPS），故障切换 |
| Base DN | 用户/组搜索基准 DN |
| 字段映射 | LDAP 属性 → 平台用户字段映射（用户唯一键 `(domain_id, username)`） |
| 同步频率 | 增量/全量同步周期；不依赖域间信任 |
| 绑定 org_id | identity_source 绑定到组织 `org_id`，各子公司管本域、集团聚合 |

- 各 identity_source **独立服务账号与密钥按域隔离存密钥库**；单域故障仅影响该域并维持权限快照。

## 六·补（V1.9 对齐 · CR-004）AI / 模型与 RAG 运行时配置

- **模型与 RAG 运行时配置**（端点 / 密钥 / RAG / 提示词 / “数据不出域”开关（默认开））经**管理后台可配**；模型 API 端点 / 密钥经密钥库注入，不落明文，配置变更入操作审计。
- 各环境模型部署方式区分：

| 环境 | 模型部署方式 |
|---|---|
| 开发 | mock / stub（不接真实模型端点） |
| 测试 | 内网模型端点（脱敏） |
| 预生产 | 接近生产部署（数据不出域开关开） |
| 生产 | 数据不出域（默认开），外部模型端点出站白名单防 SSRF |

- **代码仓库同步凭证**（建议与反馈推送 GitHub / 内网 Git）入密钥库，出站审计，可一键切内网 Git。

## 七·补（V1.9 对齐 · i18n）国际化运行时配置

- **默认语言、语言包、默认时区**为运行时可配置项（管理后台），文案外置语言包、日期/数字/时区本地化。
- 上述 i18n 配置在四环境**镜像一致**，差异仅在运行时配置值。

## 八、评审重点
配置分层与密钥管理方案；四环境一致性保障；多域控逐域隔离与连通校验；模型/RAG 与 i18n 运行时可配置项。
