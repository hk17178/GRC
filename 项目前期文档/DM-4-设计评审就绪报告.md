## 一、平台概述与建设范围

- **定位**：非银行支付机构集团一体化治理-风险-合规（GRC）平台，按子公司（含海外）严格隔离、集团聚合。
- **业务模块（M1–M11）**：制度体系 / 风险评估 / 审计管理 / 法规跟踪 / AI 智能问答 / 组织与资产 / 第三方供应商 / 权限与审批 / 看板与留痕 / 通知中心 / **监管事项(M11，新增)**。
- **独立一级菜单**：外部审计、内部审计（数据层统一 `audit_type`，界面两菜单不合并）；**合规清单**（原合规义务清单，端到端举证链）。
- **系统管理域（非业务模块）**：模型接入与管理、建议与反馈(M-FB)、系统设置。
- **技术基线**：Vue 3 + Spring Boot 3 / Flowable（嵌入）+ PostgreSQL 16（pgvector）+ Redis + MinIO；**模块化单体**（强边界 + 事件解耦 + 防腐层）。
- **排除边界（明确不建设）**：漏洞闭环管理、SIEM/扫描平台本体——GRC 仅经接口采集 KRI；IR/SOC 主体不在本平台，仅"重大事件监管报送"归口本平台。

## 三、设计质量收敛过程（四轮 workflow 审计）⭐

> 本项目设计经四轮独立多智能体审计逐层收敛，从"表面一致"深入到"功能能否跑通"。

| 轮次 | 审计层面 | 发现 | 处置结果 |
|---|---|---|---|

## 四、关键架构决策（评审重点确认项）

| # | 决策 | 取舍理由 |
|---|---|---|
| ADR-A | **风险五级 = 平台统一阈值**（模板不可改） | 保证 M9 跨子公司聚合口径一致、不失真（需求 11.4 红线） |
| ADR-B | **单域故障降级 = 安全优先**（故障域禁新登录/缩 TTL/禁敏感写；`effective_disabled=AD OR platform_disabled`） | 等保/支付场景安全优先于可用性；离职回收不被故障窗口架空 |
| ADR-C | **防篡改 = 哈希链**（operation_log/evidence curr/prev_hash 按 org 分链 + TSA） | 监管举证力强，契合等保不可篡改要求 |
| ADR-D | **数据不出域 = 条件式互斥**（开拒外部端点/关过白名单 + SSRF IP 复核） | 消除"既禁外部又过白名单"的不可判定规格；红线可判定 |
| ADR-E | **时间事件唯一生产者 = 调度/到期扫描内核** | 所有 `*_DUE/*_EXPIRING` 不漏不重，根治预警链断源 |
| ADR-F | **自由度不破隔离**：自定义查询强制注入 visibleOrgs + RLS 兜底，禁裸连/BYPASSRLS | 高自由度与组织隔离第一性并存 |

## 六、红线与合规对齐

| 红线 | 落地 |
|---|---|
| 组织隔离（子公司不互见） | 统一数据访问层注入 visibleOrgs + 全量 RLS 兜底；自定义查询同受约束 |
| 多域控（CR-003） | N 个 identity_source；`(domain_id,username)` 唯一；按域认证路由；不依赖域间信任 |
| 留痕不可篡改 | 应用账号无 UPDATE/DELETE + 哈希链 + TSA + WORM 可选 |
| AI 数据不出域 | 开关默认开 = 仅内网；外部端点出站白名单 + SSRF 复核 |
| 对外出站（反馈推送） | 脱敏 + 管理员审批 + 出站审计；可一键切内网 Git |
| 合规基线 | 等保三级 / PBOC / PIPL / ISO 27001 / PCI DSS（CHD/CDE） / ISO 27701（ROPA） |

## 七、放行结论与开发期门禁

**评审结论建议：有条件放行进入开发。** 设计层 12 条能力链路贯通、红线已对齐、文档体系完整且四轮审计收敛。

**开发期红线门禁（必须在对应里程碑前关闭，详见 D1-9 §六、D2-1 M0/M1）：**
- **M0 设计收口前**：数据模型完整 DDL 就绪（risk_finding/obligation/risk_domain/access_review/sod/M11 四台账/通知场景/自定义元数据/哈希链列/TSA）；表单引擎与隔离 PoC 通过（含多域同名/跨域隔离）。
- **M1 一期上线前（红线）**：调度/到期扫描内核、防篡改哈希链、一期外部接口确认、一期表迁移与 RLS 策略 — 必须关闭。
- **M2 二期上线前**：D1-8 高自由度低代码（生成列/动态查询编排器/流程版本快照）、混合打分权重、控件复用、模型接入。
- **M3 三期上线前**：SIEM/KRI、爬虫等三期外部接口确认。

## 九、建议评审决议

1. **确认** V1.9 需求基线与六项关键架构决策（§四 ADR-A~F）。
2. **确认** 排除边界（漏洞闭环/SIEM 本体不建设；IR/SOC 仅重大事件报送）。
3. **批准** 进入开发，并将 §七红线门禁纳入里程碑验收。
4. **指派** D1-9/D2-1 外部接口待确认项（AD/短信/企微/SIEM/模型端点/代码仓库）的对接责任人与时点。
5. **决定** 是否将 D1-8 高自由度低代码二期范围按团队产能再分批。
