## CR-004-1 新增「建议与反馈」模块（M11，系统管理）⭐

**决策：** 新增一个轻量模块，统一收集、格式化、分类、评估全体使用者的 **BUG 反馈与功能建议**，并按治理要求对外/对内同步与周期分析。

**1) 双入口收集（入站能力，新增）**
- 系统管理新增「建议与反馈」菜单：站内提交表单（类型、标题、描述、截图、所属模块、紧急度，自动带上下文：当前页面/版本/角色/子公司）。
- M5 AI 助手内可直接"提建议 / 报 BUG"，由助手结构化后落入同一反馈库。
- 说明：现有 4 类通知通道为**单向推送**（需求 12.2.1），本模块新增的是**入站采集**，不改变通道单向性质。

**2) 格式化 + 分类 + 可行性评估（独立处理）**
- 自动**格式化**为统一结构（标题/复现步骤/期望/实际/环境/附件）。
- 自动**分类**：一级 = BUG / 功能建议（可扩展：体验问题、数据问题、合规建议）；二级 = 模块归属、严重度/价值度。
- **可行性初评**：规则 + LLM 给出 价值/成本/风险 初判与建议处置（采纳/观察/婉拒），**人工复核为准**（呼应 CR-001 自由度治理：AI 建议不替代人工裁决）。

**3) 同步到代码仓库「反馈与建议」目录 —— GitHub·脱敏+审批后推送（已决策）**
- 目标：推送到项目 GitHub 仓库的 **`反馈与建议/`** 目录（或对应 Issue）。
- **合规网关（关键）：** 平台自身须满足等保三级/数据不出域，故对外推送前强制：
  1. **自动脱敏**：移除/打码 PII、密钥、内部 IP、截图敏感区、子公司可识别信息；
  2. **管理员审批**：脱敏结果经指定管理员复核确认后方可出站（默认**不自动直推**）；
  3. **出站审计**：每次推送写入平台操作审计日志（操作人/时间/内容哈希），可举证。
- 配置项：仓库地址、目录/Issue 映射、脱敏规则、审批人、推送频率；凭证（GitHub Token）入密钥管理，不落明文。
- 备选：如后续要求完全不出外网，可一键切换为**内网 Git**目标（同一推送适配器，仅换端点）。

**4) 周期性分析结论推送（通知中心）**
- M10 通知场景库新增「**反馈与建议周期分析**」场景：按周期（如每周/每月）聚合——新增/分类占比/高价值建议 Top/高频 BUG/处置进度——生成分析结论。
- 接收：指定管理员 + 指定**企业微信机器人**（沿用 M10 现有"管理层简报(聚合、不逐条打扰)"模式）。

**影响文档：** 需求 V1.9 新增 §（M11 建议与反馈）；D1-2（feedback / feedback_category / feedback_sync_log 表）；D1-4（M11 模块设计 + 脱敏审批出站流程）；D1-5（GitHub/内网 Git 推送接口，**新增一条对外出站集成**，列入集成清单与待确认项）；D1-3（脱敏/审批/出站审计/凭证）；D1-7（反馈提交页 + 分类看板 + 出站审批页规格）；M10 场景库；DM-1 RTM；DM-2 风险登记（对外出站数据外泄风险 + 缓解）。
**原型：** 系统管理增「建议与反馈」入口 + AI 助手提交（下一步可补）。

## CR-004-2 新增「AI 接入与模型管理」菜单（系统管理）⭐

**决策：** 架构层已有"模型接入层"（D1-1 §AI/RAG、D1-3 §6.4、D1-4），但**缺可视化管理入口**。新增 **系统管理 → 「AI 接入」独立一级菜单**（已决策放系统管理）。

**管理内容：**
- **模型/端点**：本地（vLLM/Ollama）或内网 API；对话模型与 **embedding 模型**分别配置；多模型可切换（可插拔）。
- **凭证**：API Key / 服务地址，入密钥管理，不明文展示。
- **RAG 参数**：召回数、相似度阈值、切片/索引刷新策略（与 D1-2 kb_chunk / pgvector 对应）。
- **提示词与防护**：系统提示词模板、提示注入防护开关、工具/数据访问范围限定在 `visibleOrgs`（D1-3 §6.4）。
- **「数据不出域」开关（默认开）**：开启时仅允许内网模型/内网 API，禁止外部模型端点；与 §6.4 敏感数据不出域一致。
- **连通性测试 + 调用审计**：测试按钮、AI 调用量/时延/失败监控（与 D4-2 运维 SOP「AI 服务异常」联动）。

**影响文档：** 需求 V1.9（M5 下增"AI 接入管理"能力，或列为系统管理配置项）；D1-2（ai_provider / ai_model_config 配置表）；D1-7（「AI 接入」配置页规格，密度 L4）；D1-5（模型接入接口标注：内网优先、不出域开关）；D1-3（密钥/不出域/提示注入治理已述，补管理面引用）；D1-4（模型接入层补管理面）；DM-1 RTM。
**原型：** 系统管理新增「AI 接入」菜单与配置页（下一步可补）。
