## 通用约定
- 分层：Controller（API）→ Application（用例编排）→ Domain（领域逻辑）→ Infrastructure（持久化/适配器）。
- 隔离上下文（visibleOrgs）在请求入口注入，贯穿到数据访问层（D1-3 §5）。
- 领域事件经事件总线发布（D1-5 §4）。

## M1 制度体系
- **职责**：制度全生命周期 + 签署确认 + 知识库构建。
- **核心流程**：起草→评审→签批（M8 审批）→发布→修订/废止；版本历史保留。
- **签署确认闭环**：发布时按 `is_core`+类型配置强制确认范围；员工在线确认→记 `policy_attestation`→入留痕；逾期未确认经 M10 提醒，未确认率入看板；MAJOR 变更触发重新确认。
- **知识库**：解析电子文档→按条款切片(`kb_chunk`)→pgvector 向量；默认检索当前生效版本。
- **事件**：`POLICY_PUBLISHED`、`ATTESTATION_OVERDUE`、`REVIEW_DUE`。
- **协作**：引用 M6 判适用范围；问答权限受 M8；切片供 M5。

## M2 风险评估
- 见 **D1-6 表单引擎专项**。补充：评估生命周期状态机 `FILLING→REVIEW→APPROVED→EFFECTIVE`；风险点一键转整改（共用工单）；KRI 监控独立配置与预警。
- **事件**：`ASSESSMENT_SUBMITTED`、`ASSESSMENT_APPROVED`、`KRI_ALERT`、`REMEDIATION_*`。

## M3 审计管理（内/外审）
- **职责**：审计计划/执行/发现/整改/报告，统一承载 **内审 + 外审 + 监管检查 + 认证审计**。
- **审计类型（`audit_type`）**：`INTERNAL`(内审) / `EXTERNAL`(外审) / `REGULATORY`(监管检查) / `CERTIFICATION`(认证审计)；按类型驱动差异化流程与视图。
- **复用**：检查表=表单引擎(usage=AUDIT)；整改=共用工单；报告=共用生成机制（D1-6 §9）。
- **差异**：检查项引用 M1 制度条款（报告自动带编号）；发现项严重度建议；趋势分析（同类问题反复）。
- **外审对外回函三段漏斗**：`已向外部机构提交 → 外方受理 → 外方确认关闭`；对外回函跟踪（提交/受理/确认时点与凭证留痕），区别于内部整改工单闭环。
- **认证有效期跟踪**：认证审计登记证书/资质有效期，到期前分级提醒（复评/再认证），有效期入看板。
- **等保测评全过程视图**：测评准备→现场测评→差距整改→出具结论的全过程专项视图（进度/差距/到期），独立于通用审计列表。
- **事件**：`AUDIT_PLAN_STARTED`、`FINDING_CREATED`、`AUDIT_REPORT_PENDING`、`EXTERNAL_RESPONSE_DUE`(对外回函时限)、`CERTIFICATION_EXPIRING`(认证到期)。

## M4 法规跟踪
- **职责**：跟踪源采集 + 变更摘要 + 法规-制度映射。
- **爬虫 Worker**：按 `regulation_source.crawl_rule` 抓取；失效检测（连续 N 次未达预期）→告警；采集规则配置化。
- **变更摘要**：条款级，结合 M5 提取变更点。
- **映射**：`regulation_policy_map` 支撑逐条合规核查。
- **事件**：`REGULATION_UPDATED`、`CRAWLER_UNHEALTHY`。

## M5 AI 智能问答
- **职责**：跨 M1/M4/M2/M3 的自然语言问答。
- **流程**：用户问→检索（**先按 visibleOrgs 过滤** kb_chunk/记录）→重排→生成（带引用）→流式返回；多轮上下文受会话隔离约束。
- **安全红线**：检索阶段过滤、复用 M8、记访问日志、防幻觉（无依据不答）（D1-3 §6）。
- **模型接入层**：可配置本地/内网模型，模型可插拔。

### M5·补 · 模型接入与管理（V1.9 对齐 · CR-004）
- **定位**：为模型接入层补**可视化管理面**，独立一级菜单挂「系统管理 → 模型接入」（产品 UI 不出现实现术语）。
- **模型注册/端点**：本地（vLLM/Ollama）或内网 API；对话模型与 embedding 模型分别注册配置；多模型可切换（可插拔）。
- **端点密钥**：API Key / 服务地址入密钥库，不落明文、不明文展示；配置变更入操作审计。
- **RAG 参数**：召回数、相似度阈值、切片/索引刷新策略（对应 D1-2 kb_chunk / pgvector）。
- **提示词与防护**：系统提示词模板、提示注入防护开关；工具/数据访问范围限定在 `visibleOrgs`。
- **「数据不出域」开关（默认开）**：运行期约束——开启时仅允许内网模型/内网 API，**禁止外部模型端点**；与 D1-3 §6.4 一致。
- **外部端点出站白名单（防 SSRF）**：当确需外部模型端点时，出站地址须命中白名单方可调用，拦截非白名单请求，调用入出站审计。
- **连通性测试 + 调用审计**：测试按钮、AI 调用量/时延/失败监控（联动 D4-2 运维 SOP「AI 服务异常」）。

## M6 组织/资产台账
- **职责**：组织树 + 资产/对象台账 + **多域控 AD 同步**。
- **多域 AD 同步适配器（CR-003）**：以「身份源（identity_source）」为单位配置 **N 个 AD 域/林**（各子公司/海外独立域），各自独立服务账号/Base DN/字段映射/同步频率并绑定 `org_id`；仍为 AD→平台单向同步；全量+增量；离职→停用→权限回收；调岗→更新 org_id；各域组织树挂其子公司 org_id 子树，集团聚合视图。
- **多域同名隔离**：用户唯一键为 **(domain_id, username)**；不依赖域间信任，同一自然人不自动合并（按需 M6 人工关联）。
- **台账**：评估对象唯一来源；供应商对象纳入。
- **作为 M1/M2/M3/M5/M8/M10 的数据底座。**

## M7 第三方/供应商
- **职责**：供应商全生命周期 + 风险评估（复用 M2）+ SLA + 事件触发复评。
- **生命周期状态机**：`ONBOARDING→IN_CONTRACT→RENEWAL→EXIT`。
- **事件触发复评**：人工登记或外部信用接口→生成复评任务（复用 M2 整改/跟踪）。
- **事件**：`VENDOR_EVENT`、`SLA_EXPIRING`。

## M8 权限与审批流
- **职责**：RBAC 四元组 + 多级审批（Flowable）+ 隔离判断 + 通知路由依据。
- **核心服务**：`PermissionService.visibleOrgs(user)`、数据访问拦截器、`AuthApprovalService`（Flowable 编排）。
- **审批**：加签/转办/超时提醒；与发起人职责分离。
- **一套权限模型全平台复用**（M5 隔离、M10 路由均依赖）。
- **多域控认证路由（CR-003）**：登录标识 `DOMAIN\user` 或 `user@domain`，按 域选择/UPN 后缀/域前缀 解析目标域并路由到对应域控校验；密码策略由各域 AD 承载，SSO 支持多 IdP（按子公司）；各子公司管本域用户与角色，集团管跨域聚合。
- **单域降级权限快照**：单域 AD 故障仅影响该域登录/同步，其余域照常；同步中断期间维持该域**权限快照**，恢复后增量补同步。

## M9 看板与留痕
- **看板**：物化视图/聚合查询，按组织/时间下钻，标注数据截止；权限受 M8。
- **留痕**：evidence（哈希+只读+保留期）、operation_log（独立 append-only）；数据库层禁改删（D1-2 §13）。
- **消费**：订阅各模块事件汇总；存 M10 发送记录。

## M10 通知中心
- **三层**：通道适配器（企微/飞书/邮件/短信）/ 规则引擎（配置）/ 分级降噪。
- **流程**：消费领域事件→规则匹配（事件×级别×组织×通道）→降噪合并/免打扰→多通道发送→失败重试→留痕。
- **联动 M6/M8** 防跨子公司广播；短信受级别控制（成本）。

## M11 监管事项
- **职责**：一级业务模块，承载监管侧合规事务：**报送日历(法定时限预警)** / **监管问询**(问询函→答复→留痕) / **处罚约谈台账** / **重大事件报送** / **年度合规计划**。
- **边界**：IR/SOC 主体不在本平台，仅「**重大事件监管报送**」在本平台落地（报送动作与留痕）。
- **组织隔离**：各子公司管理本子公司监管事项，集团聚合集团级；隔离与路由复用 M6/M8。
- **事件**：`REG_FILING_DUE`(报送时限)、`REG_INQUIRY_CREATED`、`PENALTY_RECORDED`、`MAJOR_EVENT_FILED`。

## M-FB 建议与反馈（系统管理）
- **职责**：统一收集、格式化、分类、可行性评估全体使用者的 **BUG 反馈与功能建议**，并按治理要求脱敏审批后同步代码仓库、周期分析推送。挂「系统管理」菜单（非一级业务模块，编号 M-FB；监管事项占用 M11）。
- **双入口收集（入站）**：系统管理「建议与反馈」站内提交表单（类型/标题/描述/截图/所属模块/紧急度，自动带上下文：页面/版本/角色/子公司）；M5 AI 助手内「提建议/报 BUG」结构化后落入同一反馈库。不改变现有 4 类通知通道单向推送性质。
- **格式化 → 分类 → 可行性评估**：自动格式化为统一结构（标题/复现步骤/期望/实际/环境/附件）；分类一级=BUG/功能建议（可扩展），二级=模块归属/严重度/价值度；规则+LLM 给出价值/成本/风险初评与处置建议（采纳/观察/婉拒），**人工复核为准**。
- **脱敏 + 审批后仓库同步**：对外推送（GitHub「反馈与建议/」目录或 Issue）前强制——①自动脱敏(PII/密钥/内部 IP/截图敏感区/子公司可识别信息)；②管理员审批（默认不自动直推）；③出站审计（操作人/时间/内容哈希）。凭证(GitHub Token)入密钥库不落明文；可一键切换**内网 Git**目标（同一推送适配器仅换端点，防 SSRF）。
- **周期分析推送**：M10 通知场景库「反馈与建议周期分析」按周期聚合（新增/分类占比/高价值建议 Top/高频 BUG/处置进度），推送指定管理员 + 企业微信机器人（沿用聚合简报模式，不逐条打扰）。
- **状态机**：`NEW → TRIAGED → ASSESSED → SYNCED / CLOSED`（NEW 新建 → TRIAGED 已分类 → ASSESSED 已评估 → SYNCED 已同步仓库 或 CLOSED 关闭）。
- **事件**：`FEEDBACK_CREATED`(反馈创建)、`FEEDBACK_DIGEST_DUE`(周期分析到期，供 M10)。

## 关键状态机汇总
| 对象 | 状态 |
|---|---|
| 评估 | FILLING→REVIEW→APPROVED→EFFECTIVE |
| 整改工单 | TODO→DOING→DONE→VERIFIED（超期→OVERDUE） |
| 制度 | DRAFT→REVIEW→EFFECTIVE→DEPRECATED |
| 模板版本 | DRAFT→EFFECTIVE→RETIRED |
| 供应商 | ONBOARDING→IN_CONTRACT→RENEWAL→EXIT |
| 通知发送 | PENDING→SUCCESS/FAIL→RETRIED |
| 建议与反馈(M-FB) | NEW→TRIAGED→ASSESSED→SYNCED/CLOSED |

---

## 评审重点
1. 模块边界与协作（仅经接口/事件）是否清晰、无环依赖。
2. 各状态机是否完整覆盖业务流转。
3. M5 检索隔离、M9 留痕禁改删 的落地点。

## 补充模块与能力（CR-001 / CR-002 增补）

- **新增「监管事项(M11)」一级业务模块**（详见上文 M11 章）：报送日历(法定时限预警)、监管问询(问询函→答复→留痕)、处罚/约谈台账、重大事件报送、年度合规计划；**组织隔离**：各子公司管理本子公司监管事项，集团管集团级。
- **M3 升级为「审计管理」**：统一承载内审 + **外审/监管检查/认证审计**(`audit_type`)；外审整改闭环延伸至「已向外部机构提交 → 外方受理 → 外方确认关闭」+ 对外回函跟踪与认证有效期提醒；新增**等保测评全过程**专项视图(进度/差距/到期)。
- **M7 供应商评估闭环（显式）**：准入(首次)/年度定期/续约/退出/事件触发复评 → 发现 → 整改(复用 M2 工单) → 闭环；新增**技术安全与 DPA 维度**(数据驻留/出境、PCI 范围、自身认证、再委托、连接最小权限、DPA)；定期评估计划与到期提醒。
- **M8 增**：**权限审阅(UAR)** 定期复核闭环、**越权/特权账号明细下钻**、职责分离(SoD)冲突检测、权限变更审计。
- **M9 增**：**跨子公司合规对比 benchmark**(按法规/控制点横向)、**平台自身操作审计日志**界面、**证据举证包导出 + 证据↔法规条款反向取证**、内审管理层回应/审计覆盖率。
- **M2 增**：资产-威胁-脆弱性(A-T-V)建模、**风险处置计划(降低/接受/转移/规避)+ 残余风险 + 管理层接受签认**、综合风险指数可解释下钻(构成/权重/环比归因/口径)、KRI 趋势与基线、风险**五级**。
- **M1/M4 增**：**合规义务清单(Obligation Register)** 端到端举证链(法规条款→义务→责任部门→制度/控件→证据)；**法规新增源自动分类排版**(AI 分类/条款级摘要)。
- **M5 增**：生成可报送/汇报材料、回答数据**时点标注**。
- **M6 增**：资产合规属性(数据分级/个人信息/跨境/等保定级备案/持卡人数据)、**ROPA(PIPL 法定台账)**。
- **全局**：**跨模块全文检索与溯源**、**合规态势大屏自定义组件**、**中英文(i18n)**、待办**法定时限单列预警/批量/委派代理**。

> 数据结构见 D1-2 §二十一；安全相关(UAR/SoD/越权/可信时间戳/隔离)见 D1-3；界面与下钻/趋势/检索规格见 D1-7。
