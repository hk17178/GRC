## 二、课程大纲
1. **平台概览**：GRC 是什么、解决什么问题、模块全景（见 §五·补）、信息架构、菜单顺序（外审→内审→风险→法规→制度→AI→供应商；合规态势总览/合规态势）。
2. **通用操作**：登录、导航、主题、通知、AI 助手、语言切换、全文检索、建议反馈入口。
3. **分角色实操**（按 §一 重点，配实操演练账号）。
4. **管理员专题**：表单引擎配置（克隆脚手架→结构→打分→报告→发布前测试）、控件库、通知规则、权限角色、**AI 模型管理**（模型/端点/API Key/RAG/提示词/"数据不出域"开关）、**多域控身份源**（N 个身份源、按域路由、(domain_id,username) 唯一键）、**高自由度配置**（自定义字段/视图/看板/报表/工作流/KPI/通知场景 + 发布前预览 + 变更留痕）。
5. **运维专题**：安装/升级/备份/恢复/巡检 SOP（D4-1/D4-2）。
6. **安全与合规要点**：数据隔离、留痕、签署确认的合规意义、平台操作审计日志。

## 三、培训形式与材料
- 形式：集中讲解 + 实操演练（沙箱环境）+ 答疑。
- 材料：PPT、操作演示视频、用户/管理员手册、速查卡（一页纸）。
- 考核：实操打卡 + 简单测验（关键操作）。

## 四、培训计划（配合分期上线 · 以 D2-1 开发计划/上线节奏为准）

> 培训分期严格依据 D2-1《开发计划》的分期与里程碑（一期 MVP+外审闭环、二期 AI+审计管理+监管事项+模型接入管理+反馈、三期 法规+供应商+KRI），不在本文档人为推后审计/AI/法规/供应商；各期范围如开发计划调整则以开发计划为准。

| 期 | 培训内容（对应 D2-1） |
|---|---|
| 一期上线 | 全员签署 + 评估/整改（风险五级）+ 外审回函三段漏斗 + 看板留痕 + 通知 + 多域控身份源 + 管理员配置 + i18n 语言切换 + 运维 |
| 二期上线 | AI 问答 + 知识库 + 统一审计管理（内审/外审/监管检查/认证审计 + 认证有效期 + 等保测评全过程视图）+ 监管事项（报送日历/问询/处罚约谈/重大事件报送/年度合规计划）+ AI 模型管理 + 建议与反馈 + 控件库 + UAR/A-T-V/残余风险/指数下钻 |
| 三期上线 | 法规跟踪 + 供应商 + KRI |

> 正式 PPT 与演示视频在各期上线前制作。

## 五·补（V1.9 对齐 · CR-001/002/003/004）模块全景与术语口径

> 平台概览课程使用的"模块全景"以下表为准；产品 UI 培训中不出现实现术语（M2/四元组/Flowable 等）。

**模块全景（含监管事项一级模块、统一审计管理）：**
- 制度体系管理（含签署确认、知识库问答）
- 风险评估（统一风险五级：极低/低/中/高/极高 · VERY_LOW/LOW/MID/HIGH/VERY_HIGH）+ 控件库 + KRI 持续监控
- **审计管理**（统一：内审 / 外审 / 监管检查 / 认证审计，audit_type 区分；外审对外回函三段漏斗：已提交外部机构→外方受理→外方确认关闭；认证有效期跟踪；等保测评全过程视图）
- **监管事项（一级业务模块）**：报送日历（法定时限预警）/ 监管问询 / 处罚约谈台账 / 重大事件报送 / 年度合规计划（IR/SOC 主体不在本平台，仅"重大事件监管报送"在本平台）
- 法律法规跟踪
- 第三方/供应商管理
- AI 智能问答
- 组织/资产台账（多域控身份源）+ 统一权限与审批流 + 管理看板与证据留痕 + 统一通知中心
- 系统管理：模型接入与管理、建议与反馈、高自由度配置

**菜单顺序（培训演示一致）：** 外审 → 内审 → 风险 → 法规 → 制度 → AI → 供应商；概览名称为"合规态势总览"/"合规态势"。

**统一术语口径（讲师须一致）：**
- 风险等级一律五级：极低/低/中/高/极高（不得出现"三级/高中低三级"表述）。
- 审计统称"审计管理"（不再使用"内部审计"作为模块/标题名）。
