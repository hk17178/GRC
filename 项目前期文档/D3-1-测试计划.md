# GRC 平台 · 测试计划（Test Plan）

**文档编号：** D3-1 ｜ **版本：** v1.0（草案） ｜ **日期：** 2026-06-23
**依据：** 需求 V1.8 + CR-001/002/003/004、D1-1/2/3 设计、D2-1 开发计划
**状态：** 待评审

---

## 一、引言

### 1.1 目的
定义平台的测试策略、范围、组织、进度、出入口准则与专项测试要求，确保系统功能正确、性能达标、**数据隔离零越权**、留痕可举证。

### 1.2 范围
覆盖 M1–M11 功能（含 M3 审计管理［内审/外审/监管检查/认证审计，外审三段漏斗、认证有效期、等保测评全过程视图］、M11 监管事项［报送日历/监管问询/处罚约谈台账/重大事件报送/年度合规计划］、UAR/SoD、合规卷宗/操作审计日志/benchmark、i18n、五级风险口径、多域控、模型接入与管理、建议与反馈 M-FB）、全局非功能（性能/可用性/安全/兼容）、外部集成、数据迁移。按一/二/三期分批测试，与 D2-1 里程碑对齐。

### 1.3 测试目标（可度量）
| 目标 | 标准（来源） |
|---|---|
| 功能正确 | 用例通过率 ≥ 99%，无遗留严重/致命缺陷（出口准则） |
| 数据隔离 | **跨子公司越权用例 100% 拒绝**（需求 7.3，红线） |
| 留痕可举证 | 证据/日志改删尝试 100% 被拦截（需求 11.4） |
| 性能 | 常规操作/检索 ≤3s；AI 首字 ≤5s；导出合理时间（需求 13.1） |
| 可用性 | 目标 99.5%（需求 13.2，预生产验证） |
| 安全 | OWASP 扫描 + 渗透通过，无高危（需求 13.3） |

## 二、测试策略与级别

| 级别 | 内容 | 主责 | 自动化 |
|---|---|---|---|
| 单元测试 | 关键逻辑（打分、隔离拦截、状态机、规则引擎） | 开发 | 高（CI 内） |
| 集成测试 | 模块间接口、事件、外部适配器（mock/stub） | 开发+测试 | 中 |
| 系统测试 | 端到端功能、非功能、专项 | 测试 | 部分 |
| 验收测试（UAT） | 业务部门按角色验收 | 业务+测试 | 否 |

- **按期测试**：每期（一/二/三）独立完成系统测试 + UAT 后上线；后续期做**回归**保证既有功能不被破坏（呼应"不牵一发动全身"）。
- 自动化优先覆盖：隔离/权限矩阵、表单打分、整改状态机、通知规则、回归冒烟。

## 三、测试类型与重点

| 类型 | 重点（结合本项目） |
|---|---|
| 功能测试 | M1–M11 全功能（含外审三段漏斗、监管报送日历法定时限预警、五级风险打分）；配置类（自定义字段/视图/看板/报表/工作流/KPI/通知场景）可视化操作与发布前预览/变更留痕 |
| 接口/集成 | 多域控身份路由（多 identity_source、DOMAIN\\user 与 user@domain 认证）、AD 多域同步、SIEM/日志/KRI 接口采集、短信/企微/飞书/邮件推送、爬虫、信用接口、AI 模型端点（出站白名单防 SSRF） |
| **安全测试** | 越权（水平/垂直）、注入、XSS/CSRF、文件上传、敏感泄露、**AI 越权专项**（详见 D3-3） |
| 性能测试 | 并发、检索 3s、AI 首字 5s、大批量导出异步、看板聚合 |
| 兼容性 | 公司主流浏览器版本（需求 13.4）；多主题切换显示；i18n 中英文切换与日期/数字/时区本地化显示 |
| 回归 | 每期上线前全回归；CI 冒烟 |
| 数据迁移 | Excel/Word 历史制度/评估/审计导入正确性与完整性（需求 14.3） |
| 可靠性 | 自愈、备份恢复（RTO/RPO）、降级、限流 |

## 四、测试环境与数据

### 4.1 环境
- 独立**测试环境** + **预生产环境**（容器化，与生产一致）。
- 外部系统：开发/集成期用 **mock/stub**（AD/SIEM/短信/企微等），UAT/预生产对接真实测试端点。

### 4.2 测试数据
- **多子公司样本**（≥3 家子公司 + 集团 + 部门层级），覆盖隔离测试。
- 各角色测试账号（覆盖 20 个角色 + 一人多角色组合）。
- 各体系模板样本（等保/ISO/PBOC/供应商…）。
- 敏感数据**脱敏**后用于测试；测试数据可重置。

## 五、进度与里程碑（对齐 D2-1）

| 阶段 | 测试活动 |
|---|---|
| M0 设计期 | 评审测试计划；编写用例（随设计稳定）；隔离/表单引擎/多域控路由 PoC 验证测试；五级风险口径基线确认 |

## 七、出入口准则

### 7.1 进入准则（系统测试）
- 对应期功能开发完成、自测通过；环境与测试数据就绪；冒烟通过；用例评审完成。

### 7.2 退出准则（可上线）
- 用例执行率 100%，通过率 ≥ 99%；
- **致命/严重缺陷 = 0**；一般缺陷有明确处置且不影响主流程；
- **越权专项 100% 拒绝；留痕改删 100% 拦截**（红线，不达标不上线）；
- 安全测试无高危；性能关键指标达标；
- 回归通过。

### 7.3 缺陷分级
| 级别 | 定义 | 处置 |
|---|---|---|
| 致命 | 数据越权/泄露、留痕可篡改、系统不可用、数据丢失 | 立即修复，阻断上线 |
| 严重 | 主流程不可用、核心功能错误 | 上线前必修 |
| 一般 | 非核心功能缺陷 | 计划内修复 |
| 轻微 | 体验/文案 | 可延后 |

> 数据越权与留痕篡改一律按**致命**处理。

## 八、缺陷管理流程
新建 → 指派 → 修复 → 回归验证 → 关闭；缺陷度量（密度、收敛趋势、严重分布）纳入测试报告。

## 九、专项测试纳入（强制）
1. **AI 数据隔离越权专项**（D3-3 重点）：A 子公司账号经问答/检索获取 B 子公司数据 → 全部拒绝并留日志。
2. **权限四元组边界**：角色×组织组合矩阵全覆盖。
3. **留痕不可篡改**：改/删 evidence、operation_log → 数据库层拦截。
4. **通知分级与降噪**：免打扰时段、合并推送、跨子公司不广播。
5. **爬虫失效告警**：模拟网站改版/抓取失败 → 触发 M10 告警。
6. **模板版本固化**：进行中评估不受模板后续修订影响。
7. **整改闭环**：未上传证明/未验证不可关闭。
8. **数据迁移完整性**：迁移前后核对。

### 九·补（V1.9 对齐 · CR-001/002/003/004）

9. **多域控隔离专项（CR-003）**：N 个 identity_source 各管本域；用户唯一键 (domain_id, username)；DOMAIN\\user 与 user@domain 登录均正确路由到对应域；不依赖域间信任；单域故障仅影响该域并维持权限快照，其它域与集团聚合视图不受影响；跨域用户名重名不串号。
10. **UAR/SoD 专项**：用户访问复核（UAR）周期发起、复核留痕、到期未复核告警；职责分离（SoD）冲突规则命中拦截与例外审批留痕。
11. **平台操作审计日志不可篡改专项**：平台操作审计日志（含 模型接入/模型管理配置变更、建议反馈推送、域控配置变更）改/删尝试 100% 被拦截并留痕，与合规卷宗/benchmark 证据一并校验。
12. **i18n 全覆盖专项**：中英文切换无漏译/无硬编码文案；语言包外置可热切；日期/数字/时区按区域本地化；内容双语字段正确展示；顶栏语言切换全页面一致。
13. **五级风险口径一致性专项**：极低/低/中/高/极高（VERY_LOW/LOW/MID/HIGH/VERY_HIGH）在打分、看板、报表、导出、通知文案全链路一致，无"3 级/高中低三级"残留口径。
14. **AI 数据不出域红线专项（CR-004）**：「数据不出域」开关默认开，开启时业务数据不外发外部模型端点；API Key 入密钥库不落明文；模型端点出站受白名单约束（防 SSRF）；配置变更入操作审计。

## 十、风险与度量
| 风险 | 应对 |
|---|---|
| 隔离用例覆盖不全致漏判 | 角色×组织矩阵自动化全覆盖；安全测试独立复核 |
| 外部系统不可用影响集成测试 | 用 mock/stub，预生产再真连 |
| 性能指标依赖未定的并发/数据量假设 | 设计阶段先定假设值，按假设压测并标注 |
| 测试数据敏感 | 脱敏 + 环境隔离 + 账号权限控制 |

度量指标：用例通过率、缺陷密度/收敛、越权拦截率(=100%)、关键性能 P95、覆盖率。

## 十一、评审重点
1. 出口准则（尤其越权/留痕红线 = 不达标不上线）是否认可。

3. 外部系统 mock 与真连的切换点。
4. 性能指标依赖的并发/数据量假设值确认。
