# GRC 平台 · 界面 / 交互设计规格（UI/UX Spec）

**文档编号：** D1-7 ｜ **版本：** v1.9（草案） ｜ **日期：** 2026-06-23
**依据：** D1-1 架构、D1-4 详细设计、前端设计规范（多主题令牌）
**状态：** 待评审

> **本文补齐文档体系的缺口。** D1-x 回答"系统做什么"(功能)，前端设计规范回答"长什么风格"(令牌/组件样式)，**但没有一份文档规定"每个页面具体放什么区块、怎么排布、信息密度多高"**——这正是导致原型"样样有、样样浅"的根因。本文逐页定义页面级布局与信息密度，作为高保真原型与前端开发的**唯一页面依据**。

---

## 一、设计原则

1. **信息密度优先（数据驾驶舱思维）**：合规系统是"看数据、做决策、走流程"的工具，**首屏要让用户一眼看到态势**，而非大片留白。看板/列表页应"密而不乱"。
2. **每页有明确任务**：先定义"用户来这页要完成什么"，再据此选区块，避免堆砌。
3. **区块化复用**：页面由标准"区块"(KPI 条、矩阵、卡片流、表格、工作台、详情栏…)拼装，跨页一致。
4. **栅格对齐**：12 列栅格；区块按 `span` 占列，对齐统一。
5. **空状态也要设计**："可以没有数据"≠ 空白，应有占位骨架/空状态提示，结构仍可见。

---

## 二、信息密度分级（关键标准）

| 等级 | 适用页 | 要求 |
|---|---|---|
| **L1 驾驶舱** | 合规态势、管理看板 | 首屏 ≥6 个区块拼贴：KPI 条 + 矩阵/图表 + 工作台 + 事件流；信息密度最高 |
| **L2 工作页** | 各模块列表/任务/规则/台账 | 顶部工具栏(搜索+筛选+操作) + 密实表格 + 必要的侧栏统计或概览卡 |
| **L3 详情/填写** | 制度详情、评估填写、评估报告、供应商详情 | 主内容区(70%) + 右侧元数据/进度/流程栏(30%) |
| **L4 配置/编辑** | 模板配置器、规则编辑、角色编辑、审批流编排 | 表单式 + 实时预览 |

> 反例（本次原型的问题）：把本该 L1/L2 的页面做成了"一张表 + 一个空白"，密度掉到 L3 水平。

### 3.1 顶栏主题切换器（按你的偏好改造）⭐
- **改为顶部常显命名色块**（取代下拉），5 个主题一排平铺、始终可见，更直观、一键即切：
  ```
  主题  [● 朱砂] [● 砂岩] [● 晴空] [● 苍翠] [● 宣纸]
  ```
  - 每个 = 小圆点(主题主色) + 主题名；当前项高亮(描边/底色)；

- 同步更新 **前端设计规范(样式指南)** 与 **高保真原型** 为此样式。

### 3.2 导航信息架构（主菜单 + 二级，按规定菜单顺序排列）
> 菜单顺序（统一决策）：业务区由上至下为 **外部审计 → 内部审计** → 风险评估 → 法规跟踪 → 监管事项 → **合规清单**（原合规义务清单）→ 制度体系 → AI 智能问答 → 第三方供应商。**外部审计与内部审计为两个独立一级菜单（不合并）**；二者在数据层共用 M3 审计引擎与 `audit_type`，但 UI 始终呈现为两个独立菜单。系统管理区由上至下为 通知中心 → 模型接入 → 权限与审批 → 看板与留痕 → 建议与反馈 → 系统设置。

| 分组 | 菜单 | 二级页签 / 详情页 |
|---|---|---|

| 业务 | 外部审计 | 外审任务登记｜外部机构发现｜整改跟踪(复用工单)｜对外回函三段漏斗(已提交外部机构→外方受理→外方确认关闭)｜监管检查｜认证审计与有效期跟踪｜等保测评全过程视图 |
| | 内部审计 | 内审计划｜检查表执行(引用制度条款+严重度建议)｜审计发现(含管理层回应)｜审计报告｜整改闭环｜审计覆盖率/盲区 |
| | 风险评估 | 评估任务｜模板库｜控件库｜KRI监控 → 填写页/报告页/模板配置器 |
| | 法规跟踪 | 更新流(三栏)｜跟踪源｜法规-制度映射 |
| | 监管事项 | 报送日历(法定时限预警)｜监管问询｜处罚约谈台账｜重大事件报送｜年度合规计划 |
| | 合规清单（原合规义务清单 / Obligation Register） | 义务清单(法规条款→义务→责任部门→制度/控制点→证据→满足状态)｜按来源体系分布｜按满足状态分布（**独立一级菜单**，置于监管事项与制度体系之间） |
| | 制度体系 | 台账｜知识库｜签署确认 → 制度详情 |
| | AI 智能问答 | 对话 |
| | 第三方供应商 | 台账｜SLA｜事件复评 → 供应商详情 |
| 资产与组织 | 组织与资产 | 组织架构(多身份源)｜资产台账｜ROPA |
| 系统管理 | 通知中心 | 规则｜发送记录｜通道｜企微通知场景库 → 规则编辑 |
| | 模型接入 | 模型与端点(本地/内网)｜RAG 参数｜提示词模板｜安全与不出域(数据不出域开关·出站白名单) |

## 四、标准页面区块库（拼装单元）

| 区块 | 用途 | 说明 |
|---|---|---|
| **KPI 条** | 顶部关键指标 | 一排 5–8 张窄卡：数值+趋势+迷你说明+底部进度条；密度高 |
| **热力矩阵** | 子公司×风险域 | 行=子公司，列=风险域，单元格按风险值着色(0–100)；一眼看分布 |
| **分组条形** | 整改完成率/达成度 | 横向条 + 百分比 + 逾期标记 |
| **持续监控列** | KRI/阈值 | 指标+当前值+阈值+状态点，列表式 |
| **达成度条组** | 体系合规 | ISO/等保/PBOC… 各一条，控制点覆盖率 |
| **工作台清单** | 待我审批/待办 | 类型标签 + 标题 + 责任人 + 时限(逾期红) |
| **事件流** | 实时动态 | 时间 + 类型徽标 + 摘要，竖向滚动 |
| **三栏布局** | 法规跟踪 | 左信息源、中卡片流、右概览/待办 |
| **数据表格** | 列表 | 工具栏(搜索/筛选/操作) + 密实表格 + 行内状态/进度/标签 + 可点开详情 |
| **详情栏** | 右侧元数据 | 键值对 + 进度 + 流程步骤 |
| **表单引擎** | 评估填写 | 域→控制项→题(单选/打分/文本/附件) + 跳题 + 进度/流程栏 |
| **流程图** | 审批流/生命周期 | 节点(已完成/当前/待) + 箭头 |
| **空状态** | 无数据 | 图标 + 一句说明 + 主操作按钮；结构仍在 |

## 五、逐页设计规格

> 格式：**目标 ｜ 布局 ｜ 区块清单 ｜ 密度 ｜ 关键交互**。区块"来源"指数据来自哪个模块。

### 5.0 模块主页统一密度基线（举一反三）⭐
> 总览与法规只是**示例**，密度要求适用于**所有页面**。统一原则：**每个模块的主页 = 该模块的"小型驾驶舱"**——① 模块专属 KPI 条 + ② ≥1 个把本模块数据可视化的分布/趋势图 + ③ 工作清单/密实表格 + ④ 必要的侧栏统计；**绝不做成"一张裸表 + 大片空白"**。下表是每个模块主页的区块基线（后续 5.1–5.12 在此之上细化）：

| 模块主页 | 模块专属 KPI 条 | 分布 / 趋势可视化（核心补强） | 工作清单 / 表格 | 侧栏 / 附加区块 |
|---|---|---|---|---|
| 制度体系·台账 | 总数 / 生效·草稿·评审 / 待复审 / 核心制度签署率 / 关联体系 | 按体系分布环图 · 复审到期时间轴 | 制度密实表(可点详情) | 即将到期复审清单 |
| 风险评估·任务 | 进行中 / 待审批 / 高风险 / 逾期 / 本季完成 | 风险等级分布 · 按体系&对象类型分布 · 评估进度漏斗 | 评估任务表 | 高风险待办 |
| 外部审计 / 内部审计（两个独立菜单，数据层共用 audit_type） | 年度计划 / 实施中 / 待签批 / 已归档 / 发现总数（按 `audit_type` 内审·外审·监管检查·认证拆分） | 审计进度条 · 发现按严重度·类别分布 · **同类问题复发趋势** · 整改闭环漏斗 · **外审对外回函三段漏斗** · **认证有效期跟踪** | 统一审计列表（按 `audit_type` 过滤）/ 发现表 | 待签批报告 / 等保测评全过程视图 |
| 法规跟踪 | （见 5.2 三栏） | 命中制度&映射覆盖条 | 监管动态卡片流 | 映射概览 + 待办核查 |
| 第三方供应商·台账 | 在册 / 关键 / 高风险 / SLA临期 / 待复评 | 风险评分分布 · 生命周期分布 · SLA 达标率 | 供应商表(可点详情) | 事件触发复评清单 |
| 组织与资产·资产 | 资产总数 / 按类型 / 高重要性 / **评估覆盖率** | 资产类型分布 · 评估覆盖率仪表 | 资产表（列含数据分级/个人信息/跨境/等保定级/持卡人数据/网络区域）+ ROPA 子页签 | 多身份源同步态势（各 identity_source 域，按域聚合，单域故障仅影响该域）/ 重要资产 |
| 通知中心·记录 | 今日发送 / 成功率 / 失败重试 / 紧急级量 | 送达成功率趋势 · 各通道用量 · 降噪命中率 | 发送记录表 / 规则表 | 通道连通性 |
| 权限与审批·矩阵 | 角色数 / 授权用户 / 待审批 / 超时 / **越权拦截** | 角色分布 · **权限变更审计趋势** · **UAR 复核进度** · **SoD 冲突分布** | 角色权限矩阵 / 角色表 / 权限审阅(UAR) 子页签 | 待我审批 / 审批流场景 |
| 看板与留痕·看板 | 整改完成率 / 逾期率 / 审计发现 / 通知送达率 | 风险分布 · 审计趋势 · 送达情况 | — | 留痕统计 |
| 我的待办·工作台 | 待填写 / 待审批 / 待签署 / 待整改（计数） | 待办按时限分布（逾期/今日/本周） | 分组任务列表 | — |

> 一句话：**每个模块都要有"它自己的看板"**，而不是只有总览有看板、其余都是表格。上表的"分布/趋势可视化"列就是各页相对当前原型最需要补强的部分。

### 5.1 合规态势（L1 驾驶舱）⭐对标参考版密度
> **原则（实现术语标注）**：本节及下文各区块"来源"列标注的 Mx（M2/M8/M10 等）仅为**设计标注**，用于追溯数据出处；**产品 UI 不得呈现任何实现术语**（M2/M3/M11/四元组/Flowable 等）。本页支持自定义添加组件（指标卡/图表/信息流）。
- **目标**：管理层/安全负责人一屏掌握全集团风险合规态势 + 我的待办。
- **布局**：12 列。第 1 行 KPI 条(8 卡)；第 2 行 矩阵(span6)+整改完成率(span3)+KRI监控(span3)；第 3 行 体系达成度(span4)+待我审批工作台(span4)+实时事件流(span4)。
- **区块清单**：
  | 区块 | 内容 | 来源 |
  |---|---|---|
  | KPI 条(8) | 综合风险指数 / 高风险未关闭 / 整改完成率 / KRI超阈值 / 制度签署率 / 通知送达率 / 进行中评估·审计 / 供应商高风险 | M2/M9/M10/M7 |
  | 热力矩阵 | 子公司 × 风险域(信息安全/数据合规/业务连续/第三方/监管合规/内控)，单元格风险值着色 | M2/M9 |
  | 整改完成率·分子公司 | 各子公司横条 + 逾期数 | M2 |
  | KRI 持续监控 | 漏洞修复时效/特权账号未审/日志留存/接口异常率/导出审批合规 + 阈值状态 | M2 |
  | 体系合规达成度 | ISO27001/等保/PBOC/PIPL/PCI 控制点覆盖率条 | M2 |
  | 待我审批 工作台 | 报告签批/评估确认/制度发布/复评审批，含时限逾期标记 | M8 |
  | 实时事件流 | 逾期/KRI/法规/审计/复评 徽标+摘要+时间 | M10 |
- **密度**：L1 最高，首屏铺满，不留大空白。
- **交互**：KPI/矩阵单元格/工作台项均可点击下钻到对应模块。
- **数据驱动（子公司自适应，关键）⭐**：凡以"子公司"为维度的组件——**热力矩阵的行**、**整改完成率·分子公司的条目**、**跨子公司 benchmark 的行/列**——均按 `org` 树下**实际纳入统计的子公司数量动态渲染，一子公司一行**，随子公司增减自动增减行数，不写死行数（原型中的 6 家为示例数据）。热力矩阵列（风险域）等宽均分；行数随子公司数自适应，超出视口高度时组件内纵向滚动。其余以子公司为维度的看板（M9 管理看板、benchmark、达成度对比）遵循同一规则。

### 5.2 法规跟踪 · 更新流（L2，三栏）⭐对标参考版
- **目标**：合规人员看监管动态 → 看变更摘要 → 发起逐条核查/登记映射。
- **布局**：三栏 = 左(280) 信息源清单 + 采集健康监控 ｜ 中(主) 监管动态卡片流 ｜ 右(300) 法规-制度映射概览 + 待办合规核查。
- **区块清单**：
  | 区块 | 内容 |
  |---|---|
  | 左·跟踪信息源 | 各源(央行/网信办/金融监管总局/人大法律库/证监会) + 检测频率 + 命中数 + 健康状态 |
  | 左·采集健康监控 | 正常源数 / 失效告警数 + 失效说明 |
  | 中·监管动态卡片 | 每条：来源徽标 + 紧急度 + 标题 + **AI 变更摘要(条款级)** + 命中内部制度标签(N条) + [发起逐条合规核查][登记制度映射] + 已推送 M10 标注；顶部筛选 全部/待评估/已映射 |
  | 右·映射概览 | 已建立映射数 / 待修订制度数 + 各体系映射条形(PBOC/PIPL/ISO/等保) |
  | 右·待办合规核查 | 逐条核查任务，含倒计时(剩 N 天)与具体条款 |
- **密度**：L2 偏高(中栏卡片信息丰富)。

### 5.3 制度体系（L2 + L3）
- **台账(L2)**：工具栏(搜索+体系/层级/状态筛选+新建) + 密实表格(编号/名称/体系/层级/版本/签署确认进度/状态) + 顶部 4 KPI(总数/待复审/签署率/关联体系)。行可点开**制度详情(L3)**。
- **知识库(L2)**：3 KPI(已切片/向量索引/刷新窗口) + 切片条目表(来源制度/条款/摘要/版本)。
- **签署确认(L2)**：4 KPI + 各部门签署率条形 + 未确认人员表。
- **制度详情(L3)**：主区 版本历史时间线 + 引用关系表；右栏 元数据 + 签署确认统计。

### 5.4 风险评估（L2 + L3 + L4）
- **评估任务(L2)**：任务表(编号/对象/模板/进度/风险值/截止/状态) + 顶部 KPI(进行中/待审批/高风险/逾期)。行点开 填写页或报告。
- **模板库(L2)**：体系脚手架卡片网格(8 张 + 新建)，每卡含版本/控制点数；点击进入**模板配置器(L4)**。
- **控件库(L2)**：控件表(编号/控制点/覆盖体系/复用次数/最近测试/结果)。
- **KRI 监控(L2)**：4 KPI + 指标阈值表。
- **评估填写页(L3 表单引擎)**：主区 域→控制项→题(单选/打分/文本/附件 + 跳题)；右栏 填写进度 + 提交流程步骤。
- **评估报告(L3)**：KPI(风险值/风险点/高风险/已转整改) + 风险点清单表 + 等级分布环图。
- **模板配置器(L4，建议补)**：左 结构树(域/控制项/题) + 中 题目编辑 + 右 打分/报告映射 + 顶部"发布前预览测试"。

### 5.5 外部审计 / 内部审计（L2/L3，两个独立菜单）
> 外部审计与内部审计为**两个独立一级菜单**（不合并），数据层共用 M3 审计引擎与 `audit_type`。

**5.5a 外部审计（L2/L3）**——涵盖外审 / 监管检查 / 认证审计
- **任务列表(L2)**：外审任务登记 / 外部机构发现表 / 整改跟踪(复用工单)；顶部按 外审/监管检查/认证 过滤。密度对齐 5.4。任务表字段含 编号｜认证体系｜审计机构｜责任单位｜周期｜**计划开始时间**｜状态（`plan_start_date` 必填）。
- **计划临近提醒（企微）**：可配置"计划开始前 N 天（如 15 / 10 天）"自动读取外审任务、向**指定企业微信机器人**推送提醒（外审任务·机构·计划开始日·剩余天数）；对应 M10 通知场景「外审计划临近」。
- **对外回函漏斗**：外审对外回函三段漏斗「已提交外部机构 → 外方受理 → 外方确认关闭」，每段含数量与停留时长。
- **认证审计**：认证有效期跟踪（到期预警/复评计划）。
- **等保测评全过程视图**：测评准备→现场测评→问题整改→出具报告全过程时间轴。

**5.5b 内部审计（L2/L3）**
- **统一列表(L2)**：内审计划 / 检查表执行(复用表单引擎,引用制度条款+严重度建议) / 发现表 / 报告表。密度对齐 5.4。
- **内审详情(L3)**：发现列表增 **管理层回应** 列；整改闭环漏斗；审计覆盖率/盲区。

### 5.6 第三方供应商（L2/L3）
- 台账(L2，含风险评分/SLA/生命周期) + SLA 跟踪表 + 事件复评表；详情(L3)含生命周期流程图 + 关键信息栏。

### 5.7 组织与资产（L2，含 ROPA）
- **组织架构**：左 组织树(多身份源,层级缩进) + 右 **多身份源同步态势卡**——N 个 identity_source（各子公司/海外独立 AD 域/林）按域分别展示同步状态；用户唯一键 (domain_id, username)；登录支持 `DOMAIN\user` 或 `user@domain` 认证路由；各子公司管本域、集团聚合；不依赖域间信任；单域故障仅影响该域并维持权限快照。
- **资产台账**：筛选 + 资产表，列＝编号/名称/类型/责任人/子公司/重要性/**数据分级/个人信息/跨境/等保定级/持卡人数据/网络区域**。
- **ROPA（个人信息处理活动台账）子页签**：处理活动/目的/数据类别/个人信息主体/接收方/跨境情形/留存期限；行可下钻关联资产与法规义务。

### 5.8 通知中心（L2/L4）
- 规则表 / 发送记录表 / 通道管理(4 通道状态卡 + 通道配置表)。规则编辑为 L4。

### 5.9 权限与审批（L2/L4，含 UAR/SoD/权限变更审计）
- 角色权限矩阵(C/R/U/A 网格) / 角色管理表 / 审批流配置(流程图 + 场景表)。
- **权限审阅(UAR)/SoD/权限变更审计趋势 子页签**：UAR 复核活动列表（复核进度 + 越权/特权账号明细下钻）；SoD 冲突分布与处置；**权限变更审计趋势**图（授权/回收/变更随时间）。

### 5.10 看板与留痕（L1/L2）
- 管理看板(L1)：KPI + 审计趋势 + 通知送达 + 风险分布。
- 证据留痕库(L2)：证据表(编号/关联/类型/哈希校验/入库/保留至)。

### 5.11 我的待办 · 工作台（建议新增，L2）
- 跨模块聚合：待我填写(评估) / 待我审批 / 待我签署(制度) / 待我整改；分组 Tab + 每组任务列表 + 时限。降低用户"到处找待办"。

### 5.12 AI 智能问答（L3）
- 主对话区(引用溯源 + 权限范围条) + 右栏 数据源覆盖 + 近期会话。
- **AI 助手内置「建议与反馈」入口**：可一键将当前问题转为 BUG/功能建议提交（与系统管理菜单双入口，见 5.15）。

### 5.13 监管事项（新一级业务模块，L1/L2）⭐对标参考版
- **目标**：合规团队统一管理监管报送时限、问询应答、处罚约谈台账、重大事件报送、年度合规计划。IR/SOC 主体不在本平台，仅「重大事件监管报送」在本平台。
- **五个子页签**：
  - **报送日历(L1)**：法定时限单列预警（剩 N 天/已逾期着色）+ 批量报送 + 委派；日历视图 + 临期清单 KPI。
  - **监管问询(L2)**：问询事项表（来源机构/问询内容/受理时限/应答状态/责任部门）+ 应答留痕。
  - **处罚约谈台账(L2)**：处罚/约谈记录（事由/机构/金额或措施/整改要求/闭环状态）。
  - **重大事件报送(L2)**：重大事件监管报送台账（事件/触发法规/报送时限/报送状态/回执）。
  - **年度合规计划(L2)**：计划→执行→年末对照三阶段视图 + 完成度条。

### 5.14 模型接入（原称 AI 接入；系统管理 · 独立菜单，L2/L4）⭐
- **目标**：管理员配置 AI 模型/端点/密钥/RAG/提示词，并管控"数据不出域"。
- **列表(L2)**：模型/端点表（名称/类型/端点/状态/数据不出域开关状态/最近调用）。
- **配置页(L4，表单 + 实时预览)**：
  | 配置区块 | 内容 |
  |---|---|
  | 模型 / 端点 | 模型选择、端点 URL；外部模型端点受**出站白名单**约束防 SSRF |
  | API Key / 密钥 | 密钥**入密钥库、不落明文**；界面只显示掩码与引用句柄 |
  | RAG | 数据源/索引/检索参数配置 |
  | 提示词 | 系统提示词/模板管理 |
  | 数据不出域开关 | **默认开启**；开启时禁止敏感数据外发至外部端点 |
- **留痕**：所有配置变更**入操作审计**（改端点/改密钥引用/改开关均留痕）。

### 5.15 建议与反馈（M-FB，系统管理菜单 + AI 助手双入口，L2/L4）⭐
- **目标**：收集 BUG/功能建议 → 格式化 → 分类 → AI 可行性初评（人工复核为准）→ 脱敏审批后同步到代码仓库 → M10 周期分析推送。
- **反馈提交页(L2)**：表单（类型 BUG/建议、标题、描述、复现步骤、附件）；提交后自动格式化。
- **分类看板(L2)**：按 BUG / 建议 分列看板；每卡含 AI 可行性初评标签（标注"AI 初评，人工复核为准"）、状态、责任人。
- **出站脱敏审批页(L4)**：推送到代码仓库"反馈与建议"目录前——**脱敏 + 管理员审批后推送（GitHub）**，出站审计，凭证入密钥库，支持**一键切换内网 Git**；审批界面展示脱敏前后 diff + 出站目标 + 审批人。
- **闭环**：M10 周期分析将反馈统计推送管理员 / 企微机器人。

## 六、空状态 / 加载 / 错误规格
- **空状态**：居中图标 + "暂无 XX" + 一句引导 + 主操作按钮；**保留页面骨架与区块标题**(不整页空白)。
- **加载**：骨架屏(skeleton)占位，结构先出。
- **错误**：友好提示 + 重试；敏感错误不泄露细节。

## 七、响应式与密度保障
- 目标主战场为桌面宽屏(合规工作场景)；最小宽度 1280。
- 区块按栅格 `span` 收放；窄屏时多列降为单列堆叠，但**保持密度不降级为大空白**。
- 顶部主题色块在窄屏折叠为下拉兜底。

## 八、与设计令牌的关系
- 本文只规定"放什么、怎么排、多密"；具体颜色/字体/圆角/阴影/动效**一律引用前端设计规范的令牌**(tokens.css 多主题)。
- 5 主题切换不改变页面布局与密度，只换皮。

## 九、对原型重做的验收（关键）
重做后的高保真原型须达到：
1. 总览页 ≥6 区块拼贴(含热力矩阵 + 8 KPI + 工作台 + 事件流)，首屏无大空白；
2. 法规跟踪为清晰三栏，卡片含 AI 变更摘要 + 命中制度 + 核查入口；
3. 各列表页为 L2 密度(工具栏 + 密实表格 + 侧栏统计)；
4. 主题切换器为顶部常显命名色块；
5. 全部页面/子页签/详情可点击,空页用空状态而非空白；
6. 顶栏含语言切换控件，业务菜单含**外部审计、内部审计（两个独立菜单）**、监管事项，系统管理含模型接入、建议与反馈，且 UI 不出现任何实现术语(M2/M3/Flowable 等)。

## 九·补、新增页面与区块规格（V1.9 对齐 · CR-001/CR-002/CR-003/CR-004）

**新增页面（已回填正文，索引）**

- **合规清单**（原合规义务清单，独立一级业务菜单，置于监管事项与制度体系之间）：列＝法规条款｜义务｜责任部门｜关联制度/控件｜证据｜满足状态，行可下钻端到端举证链；含按来源体系/满足状态分布。
- **ROPA**：组织/资产维度下个人信息处理活动台账。

**区块/交互增强（落到既有页）**
- **指标趋势**：KRI 与关键指标卡增 sparkline + 基线/同比；指标可下钻明细。
- **风险指数下钻**：指数卡 → 构成/权重/环比归因/口径弹层。
- **看板/热力矩阵**：单元格可下钻明细 + 趋势箭头 + 时间对比。
- **证据/卷宗**：按检查/项目一键打包导出(含清单+校验值) + 证据↔法规条款反向取证入口。
- **全文检索**：顶部检索贯通法规/制度/评估/审计/整改/证据并溯源。
- **跨子公司对比**：总览/看板按法规或控制点的子公司横向 benchmark。
- **供应商详情**：技术安全与 DPA 卡 + 评估闭环页签(准入/年度定期/续约/事件复评 → 发现 → 整改 → 闭环漏斗 + 定期评估计划)。
- **平台操作审计**：本系统 audit trail 管理界面(改阈值/导出/改制度留痕)。
- **i18n**：顶栏语言切换；五级风险标签密度。
- **大屏**：合规态势支持自定义添加组件(指标卡/图表/信息流)。
