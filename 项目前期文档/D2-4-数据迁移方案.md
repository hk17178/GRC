# GRC 平台 · 数据迁移方案

**文档编号：** D2-4 ｜ **版本：** v1.0（草案） ｜ **日期：** 2026-06-23
**依据：** 需求 V1.8 §14.3、D1-2 数据模型 ｜ **状态：** 待评审

---

## 一、目的与范围
将公司现有以 **Excel/Word** 管理的历史数据迁移导入平台：制度文件、历史风险评估记录、审计记录。

## 二、迁移对象与策略
| 对象 | 现状 | 策略 |
|---|---|---|
| 制度文件 | 已全部电子化（可编辑） | **直接解析导入**（需求 14.3）：正文存 MinIO，元数据入 policy/version，结构化切片入知识库 |
| 历史评估记录 | Excel/Word | 明确字段映射→导入 assessment（历史快照）；打分/等级尽量结构化 |
| 历史审计记录 | Excel/Word | 同上，导入 audit_plan/finding |
| 无法结构化的记录 | 零散文档 | **至少作附件留存并可检索**（需求 14.3） |

## 二·补（V1.9 对齐 · CR-001/CR-003/CR-004）新增迁移对象
以下对象一并纳入字段映射表与校验器（枚举/必填/外键/口径校验，错误行报告）：

| 对象 | 现状 | 策略 / 关键映射与校验 |
|---|---|---|
| **资产合规属性** | Excel/台账 | 导入资产台账的合规属性（数据分级、保护要求、关联合规义务、责任人、org_id）；校验数据分级枚举、责任人/组织外键 |
| **合规义务清单** | Excel/法规库导出 | 导入合规义务条目（义务来源法规、条款引用、适用范围、责任部门）；校验法规条款引用可解析、责任组织外键 |
| **ROPA（个人信息处理活动记录）** | Excel | 导入处理活动：处理目的、数据类别、跨境情形、保留期限、接收方、关联资产/供应商；校验跨境枚举、保留期限格式、关联外键 |
| **监管事项历史台账（M11）** | Excel/Word | 导入处罚台账、约谈记录、报送记录三类历史台账；校验 `event_type`（处罚/约谈/报送）枚举、法定时限/报送日期格式、监管机构字典、org_id |
| **供应商合规维度** | Excel | 导入供应商资质、DPA（数据处理协议）、跨境传输、再委托（分包）四个维度；校验资质有效期、DPA 状态枚举、跨境/再委托标志位、供应商外键 |

## 三、迁移流程
1. **盘点**：清点来源文件、数量、格式、质量。
2. **字段映射设计**：为评估/审计制定 来源列→目标字段 的映射规则（含组织归属 org_id、责任人、时间、等级）。
3. **清洗与转换**：规范化（组织/角色/等级口径统一）；缺失项标注。
4. **试迁移（小批量）**：抽样导入预生产，核对。
5. **正式迁移**：批量导入；记录迁移日志。
6. **核对**：迁移前后**数量与关键字段核对**，保留核对记录（需求 14.3）。
7. **验收**：业务抽查确认。

## 四、关键规则
- **组织归属必填**：迁移数据必须落到正确 org_id（隔离地基），缺失需人工补全。
- **保真**：保证完整性；无法结构化者转附件，不丢失。
- **不污染版本**：历史记录标记为迁移来源，与新建数据区分。

## 四·补（V1.9 对齐 · CR-003 多域控）组织归属与多域用户解析
- **按子公司域映射 org_id 子树**：迁移数据按来源子公司所属 `identity_source`（域）映射到对应 org_id 子树，确保隔离边界与域边界一致。
- **用户历史按 (domain_id, username) 解析**：历史记录中的责任人/操作人按用户唯一键 `(domain_id, username)` 解析；**多域同名（同一 username 出现在不同域）不自动合并，需人工关联确认**，避免跨域错挂。
- **多域历史归属歧义**：风险等历史记录若来源域不可判定或责任人跨域歧义，标注为"待人工关联/待定域"，不强制落库，由各子公司本域审核或集团统一裁定后再导入。

## 五、工具
- 导入模板（Excel）+ 校验器（字段/枚举/必填校验，错误行报告）；
- 制度文档解析器（结构化切片，复用 M1 知识库构建）。

## 六、回退
迁移在独立批次进行，失败可按批次回滚；正式迁移前预生产演练。

## 七、风险
| 风险 | 应对 |
|---|---|
| 历史数据口径不一致（等级定义不同） | 迁移前统一口径映射（呼应看板口径统一）；按"五级统一映射规则"归一（见 §九·补） |
| 组织归属缺失 / 多域同名歧义 | 人工补全；多域同名需人工关联，缺失不导入 |
| 大文件/海量附件 | 分批 + MinIO 容量预估 |

## 八、评审重点
字段映射规则、组织归属补全方案、核对与验收标准。

## 九·补（V1.9 对齐 · 风险五级统一）历史风险等级映射规则
平台风险等级统一为五级：**极低 / 低 / 中 / 高 / 极高（VERY_LOW / LOW / MID / HIGH / VERY_HIGH）**。历史记录可能使用旧的"高/中/低三级"或其它自定义口径，迁移时按下表归一，并**强制人工复核**确认每条映射结果：

| 历史口径（示例） | 映射到五级 |
|---|---|
| 低 / Low / L | 低（LOW） |
| 中 / Medium / M | 中（MID） |
| 高 / High / H | 高（HIGH） |
| （旧三级中无对应的极端项，按打分阈值判定） | 极低（VERY_LOW）/ 极高（VERY_HIGH） |

- **打分优先**：若历史记录含量化分值（如 LxI 矩阵得分），优先按平台五级阈值重新计算落级，等级文字仅作辅助校验。
- **人工复核为准**：三级→五级存在信息丢失（无法机械区分"极高 vs 高""极低 vs 低"），所有映射结果须经业务人工复核后方可定级；存疑项标注"待复核"，复核前不参与态势统计。
- 迁移来源等级与归一后等级**双留痕**，保留追溯。

## 十·补（V1.9 对齐 · 法规/制度/义务关联与 i18n）映射关系迁移
- **历史法规—制度条款映射迁移**：迁移历史维护的"法规条款 ↔ 制度条款"对应关系，导入条款级映射表；校验法规条款引用、制度条款编号均可解析且外键有效。
- **义务—制度关联迁移**：迁移"合规义务 ↔ 制度（落实条款）"关联，建立义务覆盖关系；校验义务条目、制度版本外键有效，标记覆盖/部分覆盖/未覆盖状态。
- **双语制度按 i18n 内容字段迁移**：制度若存在中英文双语正文，按 i18n 设计将各语言版本写入对应的语言内容字段（如 `content_zh` / `content_en` 或多语 JSONB），而非各自建独立记录；保留语言标识与默认展示语言，缺失语种留空不臆造译文。
