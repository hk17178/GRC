# GRC 平台 · 接口设计文档

**文档编号：** D1-5 ｜ **版本：** v1.0（草案） ｜ **日期：** 2026-06-23
**依据：** 需求 V1.8（§十五 外部系统集成、各模块集成点）、D1-1 架构、D1-3 安全
**状态：** 待评审

> 本文规范平台的**内部 API** 与**外部系统集成接口**。所有外部对接经**防腐层适配器**封装（D1-1 §4.5），外部变更影响范围可控。

---

## 一、接口分类
1. **内部 API**：前端 ↔ 后端（REST）。
2. **外部集成接口**：平台 ↔ AD/SIEM/日志/漏洞/短信/企微/飞书/邮件/法规网站/信用接口。
3. **领域事件**：模块间解耦（发布/订阅，供 M10/M9 消费）。

---

## 二、内部 API 规范

### 2.1 风格与约定
| 项 | 约定 |
|---|---|
| 风格 | RESTful，资源化 URL；`/api/v1/...`（**版本化**，便于演进，D1-1 §12） |
| 认证 | 会话/Token（基于 AD 登录）；每请求携带身份 |
| 鉴权与隔离 | 服务端强制校验 RBAC 四元组；自动注入 `visibleOrgs` 隔离上下文（D1-3 §5），前端不可绕过 |
| 分页 | `page`/`size`，返回 `total` |
| 统一响应 | `{ code, message, data, traceId }` |
| 错误码 | 业务错误码分段（如 4xxxx 业务、5xxxx 系统）；不泄露敏感信息 |
| 幂等 | 写操作支持幂等键（防重复提交） |
| 审计 | 敏感操作记 `operation_log` |

### 2.2 典型资源（示例，完整清单随 D1-4 细化）
```
GET   /api/v1/assessments?org=&status=&page=        评估列表（自动隔离过滤）
POST  /api/v1/assessments                            发起评估
GET   /api/v1/assessments/{id}                        评估详情
POST  /api/v1/assessments/{id}/submit                提交（进入审批流）
GET   /api/v1/templates / {id}/versions              模板与版本
POST  /api/v1/templates/{id}/versions/{v}/preview    发布前预览测试
GET   /api/v1/policies / {id}/attestations           制度与签署确认
POST  /api/v1/remediations/{id}/verify               整改验证
GET   /api/v1/dashboard/overview?org=&period=        看板聚合
POST  /api/v1/ai/chat                                 AI 问答（流式；隔离过滤）
GET   /api/v1/notifications/rules                     通知规则（配置）
```

#### 2.2 补（V1.9 对齐 · CR-004）模型接入管理 + 建议与反馈内部 API

> 以下内部 API 服务于"模型接入与管理"（系统管理独立菜单）与"建议与反馈"（M11）两模块；UI 不出现实现术语。

**模型接入与管理（CR-004-2）**
```
GET   /api/v1/ai/models                               模型/端点列表（对话/embedding 分类）
POST  /api/v1/ai/models                               新增模型/端点配置（API Key 入密钥库，不落明文）
PUT   /api/v1/ai/models/{id}                           修改模型/端点配置
DELETE /api/v1/ai/models/{id}                          删除模型/端点配置
POST  /api/v1/ai/models/{id}/test                      连通性测试（按钮触发；返回时延/可用性）
GET   /api/v1/ai/rag-config                            RAG 参数（召回数/相似度阈值/切片/索引刷新策略）
PUT   /api/v1/ai/rag-config                            修改 RAG 参数
GET   /api/v1/ai/prompts                               系统提示词模板与提示注入防护开关
PUT   /api/v1/ai/prompts                               维护提示词模板/防护开关
GET   /api/v1/ai/data-residency                        「数据不出域」开关状态（默认开）
PUT   /api/v1/ai/data-residency                        切换「数据不出域」开关
```
- 约定：API Key/服务地址入密钥管理，响应**不回显明文**；所有配置变更写入操作审计；`data-residency=开` 时仅允许内网模型/内网 API，外部模型端点出站受白名单约束（防 SSRF，D1-3 §9/§6.4）。

**建议与反馈（M11 · CR-004-1）**
```
POST  /api/v1/feedback                                提交反馈（BUG/建议；自动带上下文）
GET   /api/v1/feedback?type=&module=&status=&page=    反馈列表（隔离过滤）
GET   /api/v1/feedback/{id}                            反馈详情（含 AI 可行性初评，人工复核为准）
PUT   /api/v1/feedback/{id}/category                  人工复核分类/处置（采纳/观察/婉拒）
POST  /api/v1/feedback/{id}/desensitize               生成脱敏结果（PII/密钥/内部 IP/可识别信息）
POST  /api/v1/feedback/{id}/approve-sync              管理员审批后推送代码仓库（出站审计）
GET   /api/v1/feedback/sync-logs                       出站推送审计日志
```
- 约定：对外推送默认**不自动直出**，须脱敏 + 管理员审批后出站；每次推送写出站审计（操作人/时间/内容哈希）；仓库凭证入密钥库，可一键切内网 Git。

### 2.3 AI 问答接口（特别约定）
- 流式返回（首字 ≤5s，需求 13.1）；
- 服务端在**检索阶段**按 `visibleOrgs` 过滤（安全红线，D1-3 §6），不在生成后过滤；
- 响应含**引用来源**（制度条款编号/记录编号）；涉敏感访问记日志。

## 三、外部集成接口（按需求 §15）

> 通用要求（适用全部）：标准化接口（REST/Webhook）封装 + 异常（超时/认证失败/格式异常）容错与告警 + 凭证安全存储（不明文）+ 采集类明确频率与格式，由相关系统运维团队共同确认接口规范。

### 3.1 AD（Active Directory）— 用户/组织同步
| 项 | 内容 |
|---|---|
| 用途 | 同步用户账号、组织架构、部门归属（身份与权限基础） |
| 方向 | AD → 平台 |
| 协议 | LDAP/LDAPS（或 AD 同步 API） |
| 数据 | 用户(username/display/email/ad_object_id/部门)、组织树 |
| 模式 | 全量初始化 + **增量同步**（基于变更游标/ad_object_id） |
| 频率 | **待定**（设计阶段定值，需求 8.3/10.4）；离职/调岗时效需明确 |
| 异常 | 同步失败告警；不覆盖式更新（保留审计） |
| 安全 | 服务账号最小权限 + 凭证入密钥库 |
| **多域控（CR-003）** | **支持配置 N 个身份源（各子公司/海外独立 AD 域/林）**，每个域独立 LDAP(S) 地址（可多控制器）、服务账号、Base DN、同步频率，绑定对应 `org_id`；单域不可用不影响其余域。认证按 `DOMAIN\user` / `user@domain` 路由到对应域控；账号唯一性为"域内唯一"。详见 D1-3 §3.1.1、D1-2 §四。 |

### 3.2 SIEM / 日志平台 / 漏洞管理 — KRI 数据采集
| 项 | 内容 |
|---|---|
| 用途 | 采集 KRI 指标（日志留存天数、漏洞修复时效、接口异常率等，需求 4.2.5） |
| 方向 | 外部 → 平台 |
| 模式 | 平台定时**拉取**（REST/查询）或外部**推送**（按系统能力） |
| 频率/格式 | **待与运维共同确认**（需求 4.2.5/§15 集成通用要求） |
| 异常 | 采集失败/数据缺失告警；指标缺失不误判 |

### 3.3 短信网关（公司自有）— 紧急级通知
| 项 | 内容 |
|---|---|
| 方向 | 平台 → 短信平台 |
| 用途 | 紧急级通知（成本敏感，由规则引擎控制是否启用） |
| 约束 | **调用额度限制**需提前确认（需求 12.4），防超额；失败重试 |
| 安全 | 密钥入密钥库 |

### 3.4 企业微信 / 飞书 — Webhook 群机器人
| 项 | 内容 |
|---|---|
| 方向 | 平台 → 企微/飞书 |
| 用途 | 单向推送通知（**不要求接收交互式回复**，需求 12.2.1） |
| 接入 | Webhook 群机器人即可，无需自建应用 |
| 约束 | 单条消息长度限制、推送频率限制（防限流/封禁，需求 12.1）；机器人被移出群/地址失效需检测告警 |
| 内容 | 可附跳转外链直达任务详情页（外链打开平台，非通道内操作，需求 12.2.4） |

### 3.5 邮件系统（SMTP）
| 项 | 内容 |
|---|---|
| 方向 | 平台 → 邮件系统 |
| 用途 | 邮件通知 + 留痕备份（重要通知落一份邮件） |
| 配置 | SMTP 服务器/账号（安装向导可填，可测试发送） |

### 3.6 法规网站 — 自建爬虫
| 项 | 内容 |
|---|---|
| 方向 | 法规网站 → 平台 |
| 用途 | 采集央行/网信办/金融监管总局等发布内容（需求 6.3） |
| 模式 | 独立爬虫 Worker，按源配置的采集规则抓取 |
| **采集后处理** | **抓取 → AI 分类（法规类别/适用范围）+ 条款级摘要 → 结构化入卡片流**（供法规模块卡片化呈现与追溯）；AI 结果为辅助，人工复核为准。 |
| 配置化 | 采集规则可配，网站改版改配置而非改码 |
| 健康监控 | 连续多次未抓到预期内容→触发 M10 告警运维（需求 6.3） |
| 合规 | 评估反爬/使用条款限制，确保采集合规与技术可持续（需求 6.3） |
| 安全 | 目标白名单，防 SSRF（D1-3 §9） |

### 3.7 企业信用查询接口（可选）— 供应商事件触发
| 项 | 内容 |
|---|---|
| 方向 | 外部 → 平台 |
| 用途 | 供应商外部风险事件自动触发复评（需求 9.2）；亦支持人工登记 |
| 模式 | 接口轮询/订阅；可选 |

**大模型推理服务端点**
| 项 | 内容 |
|---|---|
| 方向 | 平台 ↔ 模型端点（请求/流式响应） |
| 用途 | 对话模型与 embedding 模型推理（RAG/AI 问答/反馈可行性初评/法规分类摘要） |
| 端点 | 本地/内网优先（vLLM/Ollama/内网 API）；外部模型端点需出站白名单（防 SSRF，D1-3 §9） |

| 安全 | API Key 入密钥库不明文；连通性测试；调用量/时延/失败审计监控 |

**代码仓库（GitLab/GitHub，可选）— 反馈与建议出站**
| 项 | 内容 |
|---|---|
| 方向 | 平台 → 代码仓库（`反馈与建议/` 目录或 Issue） |
| 用途 | M11 建议与反馈对外/内同步（CR-004-1） |
| 出站网关 | **自动脱敏 + 管理员审批后**方可出站，默认不自动直推；每次推送写出站审计（操作人/时间/内容哈希） |
| 连通性 | 推送前连通性测试；可一键切换内网 Git（同一推送适配器，仅换端点） |
| 安全 | 仓库凭证（Token）入密钥库不明文；目标地址白名单 |

## 四、领域事件接口（模块解耦）

- 生产者发布领域事件到事件总线（Redis Stream），消费者（M10 通知、M9 看板）订阅。
- 事件 Schema（统一信封）：
```json
{
  "eventId": "uuid",
  "type": "REMEDIATION_OVERDUE",          // 评估完成/整改逾期/法规更新/制度发布/KRI预警…
  "sourceModule": "M2",
  "orgId": 120,
  "occurredAt": "2026-06-23T08:00:00Z",
  "payload": { "taskId": 392, "assignee": 11, "overdueDays": 2 }
}
```
- **新增事件类型或新增订阅者，不改动生产者**（D1-1 §4.3，不牵一发动全身）。
- 通知路由结合 M6 组织 + M8 权限，确保不跨子公司广播（需求 12.2.2）。

#### 四·补（V1.9 对齐 · CR-002/CR-004）事件清单补充

| 事件类型 | 来源 | 说明 |
|---|---|---|
| `EXT_AUDIT_RESPONSE_SUBMITTED` | M3 审计管理 | 外审对外回函——已提交外部机构（三段漏斗第一段） |
| `EXT_AUDIT_CLOSED` | M3 审计管理 | 外审对外回函——外方确认关闭（三段漏斗终段） |
| `REG_FILING_DUE` | M11 监管事项 | 报送日历法定时限预警（报送临期/逾期） |
| `COMPLIANCE_DIGEST_DUE` | M10 | 年度合规计划/合规态势周期简报到期，聚合推送 |
| `FEEDBACK_DIGEST_DUE` | M11 建议与反馈 | 反馈与建议周期分析到期，推送管理员/企微机器人（CR-004-1） |

## 五、防腐层与容错（通用机制）
- 每个外部系统一个**适配器**，实现统一内部接口；外部改版/替换只改适配器。
- 统一容错：超时控制、重试（指数退避）、熔断、降级；失败多次告警管理员（复用 M10）。
- 限流：对外部接口（短信额度、企微/飞书频率）限流，避免被限流封禁。
- 凭证：统一密钥管理，不明文（D1-3 §10）。

---

## 六、接口安全
- 内部 API：认证 + RBAC 四元组 + 隔离上下文注入；输入校验；输出脱敏。
- 集成接口：凭证安全、最小权限、传输 TLS、调用审计。
- 爬虫：目标白名单防 SSRF。

---

## 七、评审重点
1. 外部接口的**频率/额度/格式**待确认项清单及责任人（AD/SIEM/短信/企微等）。
2. 内部 API 的隔离上下文注入与版本化策略。
3. 领域事件 Schema 与总线选型（Redis Stream 起步，可换 RabbitMQ/Kafka）。
4. AI 接口检索阶段过滤的接口契约。

> 配套：D1-4 详细设计将细化各模块完整 API 清单；D4-1 安装/部署将把外部接口参数纳入 Web 向导配置项。
