## 1. 认证与会话（增强③ R1）
- [ ] 未登录直接访问 `/#/dashboard` → 自动跳登录页
- [ ] 错误口令 → 提示"用户名或口令错误"，不进入
- [ ] 正确口令 → 进仪表盘，右上角显示登录人 + 退出
- [ ] 退出 → 回登录页；再访问受保护页仍跳登录
- [ ] 会话失效（清 Cookie）后操作 → 跳登录

## 2. 数据隔离（RLS）
- [ ] pay_user 仅见 org12 数据；cf_user 仅见 org13；group_admin 见全部
- [ ] 任一页的列表/看板/留痕均不跨组织泄露

## 3. 功能级 RBAC 三级权限（增强③ R2–R6）
- [ ] cf_user 登录 → 左菜单仅 11 项（系统设置/权限/审批流配置/模型接入/审计/组织资产/反馈 等**隐藏**）
- [ ] cf_user 进"制度发布"(RO) → "新建制度"按钮**置灰**；风险评估(RW) → "发起评估"**可用**
- [ ] cf_user 直接 POST 受限写接口（如法规/供应商/设置）→ 后端 **403**（绕过前端亦挡）
- [ ] group_admin 进"权限配置" → 选角色 → 矩阵正确回显（读写绿/只读琥珀/隐藏灰），改后保存生效
- [ ] group_admin 进"权限与审批" → 给某用户授/撤角色（下拉含全部角色：超管/风险专员/自定义）
- [ ] 新建一个自定义角色 → 配若干菜单/按钮 → 授给某用户 → 该用户登录后菜单/按钮按配置生效

## 4. 审批流可配置（增强②）
- [ ] "审批流配置"画布：拖节点(审批/条件/并行/结束)、连线、属性面板(会签/或签/人数/超时升级)
- [ ] 保存草稿 → 发布生效（后端编译 BPMN 部署，提示"已发布并生效"）
- [ ] 串行/会签(全过)/或签(任一过)/并行/条件/超时升级 各跑一遍（可借后端已验证范式）
- [ ] "我的待办"→"我的审批"：登录人按其角色看到分给自己的待办审批（cf_user 与 group_admin 不同）

## 5. AI 智能问答（增强①+C）
- [ ] "AI 智能问答"：录入一篇制度/法规 → 提问 → 返回回答 + 引用（本地离线模式诚实标注）
- [ ] "模型接入"：显示当前模式（本地离线/Claude/openai）+ 切换指引；界面无密钥录入框
- [ ] （可选·需密钥）部署侧设 GRC_AI_PROVIDER=claude/openai + 密钥 → 重启 → 生成式回答

## 6. 各业务模块红线（抽验）
- [ ] 风险评估：残余高/极高且无风险接受 → "关闭"被门控；申请接受→审批通过→放行（CR-002）
- [ ] 制度发布：草稿→提交评审→审批通过才生效（Flowable 审批两步）
- [ ] 监管报送：准备→提交评审→审批通过→已报送
- [ ] 第三方供应商：未评估不得启用（准入门控）
- [ ] 合规清单：标记落实须填证据（落实证据红线）
- [ ] 权限/SoD：授予互斥角色被 SoD 阻断（红框显后端消息）→ 申请例外→审批通过→放行
- [ ] 看板与留痕：哈希链校验"链完整·已校验N条"；任一写操作后留痕新增
