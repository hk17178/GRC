## 一、安装前准备（一次性）

### 1.1 服务器要求（单机一体化形态）
| 项 | 最低 | 推荐 |
|---|---|---|
| 操作系统 | CentOS 7+/Ubuntu 20.04+/麒麟 V10 等主流 Linux | 同左 |
| CPU | 4 核 | 8 核 |
| 内存 | 8 GB | 16 GB（含 AI 则 32 GB） |
| 磁盘 | 100 GB | 500 GB+（证据材料增长，见 D4-2 扩容） |
| 网络 | 内网可达即可 | — |

### 1.2 前置软件
仅需 **Docker** 与 **Docker Compose**。检查是否已装：
```bash
docker --version        # ⌨️
docker compose version  # ⌨️
```
✅ 看到版本号即可。未安装见附录 A（一条命令安装 Docker）。

### 1.3 获取安装包
- **在线环境**：可直接拉取镜像（见 2.1）。
- **离线环境（金融/政务内网常用）**：从制品库下载离线包 `mandao-grc-offline.tar` 与 `mandao-grc-deploy.zip`，传至服务器。

## 二、安装（一条命令启动）

### 2.1 在线安装
```bash
# 方式一：一键脚本（自动下载编排文件与镜像并启动）
curl -fsSL https://<内网制品库地址>/install.sh | bash   # ⌨️

# 方式二：已获取部署包
unzip mandao-grc-deploy.zip && cd mandao-grc            # ⌨️
docker compose up -d                                     # ⌨️
```

### 2.2 离线安装（无外网）
```bash
# 1) 导入离线镜像
docker load -i mandao-grc-offline.tar                    # ⌨️
# 2) 解压部署包并启动
unzip mandao-grc-deploy.zip && cd mandao-grc             # ⌨️
./install.sh                                              # ⌨️（自动启动全栈并等待依赖就绪）
```

### 2.3 确认服务启动
```bash
docker compose ps         # ⌨️
```
✅ 所有服务 `State` 为 `Up`/`healthy`（nginx、grc-app、postgres、redis、minio 等）。
⚠️ 首次启动需等待 1–3 分钟初始化，`postgres` 就绪后 `grc-app` 才会 healthy。

## 三、Web 安装向导（首次配置，浏览器操作）

打开浏览器访问：`http://<服务器IP>:8080/setup`

按向导**逐步**填写（每步有"测试/校验"按钮，失败会提示原因）：

| 步骤 | 操作 | 说明 |
|---|---|---|
| ① 环境自检 | 点"开始自检" | ✅ DB/Redis/MinIO 连通、端口、磁盘均通过 |
| ② 数据库 | 用内置 PG（默认，推荐）或填外部 PG 连接 → "测试连接" | 内置即开箱可用 |
| ③ 初始化 | 点"初始化" | 自动建库、执行迁移、灌入内置体系脚手架（等保/ISO/PBOC…） |
| ④ 管理员 | 设置平台超级管理员账号/密码 | 首个登录账号 |
| ⑤ 身份源配置 | **逐个添加 N 个身份源**（各子公司/海外独立 AD 域或林）：每个填 LDAP(S) 多控制器地址（HA 故障转移）、独立服务账号密钥、Base DN、字段映射（domain_id/username/邮箱/组织等）、同步频率，并**绑定 org_id**；逐个点"测试连接/测试同步"（可跳过，后续在后台配） | 登录标识支持 `user@domain`(UPN) 或 `DOMAIN\user`；各身份源分别同步用户/组织 |
| ⑥ AI 模型接入（可选） | 配置模型/端点/API Key/RAG、开启"数据不出域"开关（默认开）→ "测试连接"（可跳过，后续在"系统管理 → AI 接入与模型管理"后台配） | 密钥加密入密钥库不落明文；外部端点受出站白名单约束 |
| ⑦ 通知通道 | 填企微/飞书 Webhook、邮件 SMTP、短信网关 → 各"测试发送" | 逐个验证连通 |
| ⑧ 监管参数 | 确认留痕保留期限等参数 | 取各监管口径最大值 |
| ⑨ 完成 | 点"完成安装" | 写入配置、标记已安装、跳转登录页 |

✅ 进入登录页即安装成功。用 ④ 设置的管理员账号登录。
⚠️ 完成后 `/setup` 自动关闭，防止被再次初始化。
⚠️ 多身份源说明：各身份源相互独立、**不依赖域间信任**；集团侧聚合视图、各子公司管理本域；单个身份源（域）故障**仅影响该域**登录与同步，其它域不受影响并维持已有权限快照。

## 四、安装后验证（冒烟）
1. 用管理员登录，左侧菜单可正常切换各业务模块——按菜单顺序至少含：**外审 / 内审**（审计管理：内审/外审/监管检查/认证审计）、**风险**、**法规**、**制度**、**AI**、**供应商**，以及**监管事项**（报送日历/监管问询/处罚约谈/重大事件报送/年度合规计划）入口均可正常打开；
2. 进入"系统管理 → 通知中心"，发一条测试通知，企微/飞书/邮件能收到；
3. 进入"风险评估 → 模板库"，确认内置脚手架已存在；
4. 右上角切换主题、切换语言（中/英）正常；
5. （若配了身份源）"系统管理 → 组织台账"能看到**各身份源分别同步**的用户/组织；
6. （若配了 AI）进入 AI 助手发一条问答，确认模型端点连通、"数据不出域"开关状态正确。

## 五、高可用集群形态（生产规模，可选）

同一套镜像，改用集群编排（`docker-compose.ha.yml` 或 K8s）：
- `grc-app` 多副本 + Nginx 负载；
- PostgreSQL 主备 + 自动故障转移；
- Redis 哨兵；MinIO 多盘；
- 外置配置/密钥库。

⚠️ 集群形态由运维/DevOps 按《D4-2 运维 SOP》§集群部署执行；中小规模/试点用单机形态即可。

## 六、升级与回滚（详见 D4-2）
```bash
./upgrade.sh             # ⌨️ 拉新镜像→自动备份→DB迁移→滚动重启→健康校验
./rollback.sh <版本号>   # ⌨️ 回滚镜像 + 数据库回滚点
```
⚠️ 升级前会自动备份；离线环境升级前请先导入新版离线镜像。

## 七、目录与持久化数据（勿误删）
```
mandao-grc/
  docker-compose.yml      编排文件
  config/                 配置（向导写入；身份源服务账号密钥、AI 模型端点与 API Key 等凭证均加密存储入密钥库，不落明文）
  install.sh upgrade.sh rollback.sh backup.sh restore.sh doctor.sh
volumes（持久化，升级不丢）：
  pgdata/                 数据库数据
  miniodata/              附件与证据材料
  logs/                   运行/审计日志
```
⚠️ 卸载/迁移服务器时，备份并迁移上述 volumes 即可保留全部数据。

## 八、常见问题（FAQ / 故障速查）
| 现象 | 排查 |
|---|---|
| `grc-app` 一直 unhealthy | 等 PG 就绪；`docker compose logs grc-app` 看错误；检查 ③ 是否初始化 |
| 向导 ① 自检失败 | 端口被占用？磁盘满？`docker compose ps` 看依赖是否 Up |
| 通知测试发送失败 | 核对 Webhook 地址/SMTP/短信密钥；网络是否可达外部 |
| 某身份源同步失败 | 按**该身份源**逐项排查：服务账号密钥/权限、LDAP(S) 控制器地址（多控制器是否均不可达）、Base DN 与字段映射、防火墙；单个身份源不可用**仅影响该域**，其它域照常，集团聚合视图自动剔除该域 |
| AI 问答无响应/连通失败 | 核对模型端点地址、API Key（是否已入密钥库）、出站白名单是否放通该端点；"数据不出域"开启时仅允许内网模型 |
| 页面打不开 | Nginx 是否 Up；端口 8080 是否放通 |
| 一键体检 | `./doctor.sh` 一次性检查各组件/磁盘/外部连通/备份状态 |
