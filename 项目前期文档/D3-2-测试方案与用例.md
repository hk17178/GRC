## 一、用例设计方法与规范

- **方法**：等价类 + 边界值 + 场景法 + 决策表（权限/规则类）+ 状态迁移（整改/审批）。
- **编号**：`TC-<模块>-<序号>`，如 `TC-M2-008`；关键安全场景 `TC-SEC-xxx`。
- **要素**：编号 / 模块 / 标题 / 优先级 / 前置条件 / 步骤 / 预期结果 / 关联需求。
- **优先级**：P0 红线（隔离/留痕/闭环）｜P1 核心｜P2 一般。

### 11.1 风险五级与建模（CR-001-7 / CR-002）

#### TC-M2-104 五级 levelMatrix 平台统一阈值基线 + 边界值 + risk_level 枚举（P0）
| 项 | 内容 |
|---|---|
| 模块/优先级 | M2/M9 ｜ **P0** ｜ 关联 4.2.2、CR-001-7、D1-6、A1 |
| 前置 | 平台级统一 `levelMatrix` 基线已配置（极低/低/中/高/极高 对应固定 risk_value 区间）；存在两套不同模板 T1/T2 与至少两家子公司 ORG_A/ORG_B |
| 步骤 | 1) 取各档分值边界（上下界、临界±1、量程首尾）打分，校验落档（`value≤max 归本档`）与 risk_level 枚举值 VERY_LOW/LOW/MID/HIGH/VERY_HIGH；2) 在模板 T1/T2 编辑界面尝试修改 `levelMatrix` 阈值区间；3) 用 ORG_A 与 ORG_B 对**同一 risk_value** 落档比对；4) 在 M9 看板/报表按五级聚合，并做 M9 跨子公司 `risk_level` 计数聚合；5) 构造"有洞/重叠/非单调"的 levelMatrix 尝试保存（求值校验：相邻 max 升序连续、单调递增、全量程覆盖） |
| 预期 | 各边界落档与五级一致、无"3 级/高中低三级"残留、枚举值统一；2) 模板**不可改阈值**（`levelMatrix` 为平台级统一基线、模板侧只读/禁改）；3) 不同模板/不同子公司对同一 risk_value 落档**完全一致**；4) 看板五级配色/统计正确，M9 跨子公司按 `risk_level` 计数语义一致、**不再失真**、可横向对比；5) 有洞/重叠/非单调的矩阵保存被校验拒绝 |

#### TC-M2-106 风险点（risk_finding）处置 + 残余风险 + 管理层接受=关闭门控（P0）
| 项 | 内容 |
|---|---|
| 模块/优先级 | M2 ｜ **P0** ｜ 关联 4.2.4、CR-002、C3、F2 |
| 前置 | 评估提交后物化出 `risk_finding`（承载 inherent_level/treatment_decision/residual_level/risk_acceptance）；`remediation_task` 在 source_type=ASSESSMENT 时 source_id 指向 `risk_finding.id` |
| 步骤 | 1) 对高/极高风险点在 `risk_finding` 级录入 `treatment_decision`（MITIGATE/ACCEPT/TRANSFER/AVOID）；2) 录入 `residual_level` 并重新评五级；3) 残余为 HIGH/VERY_HIGH 时，**不录入** `risk_acceptance` 直接驱动关联 remediation_task 走 DONE→VERIFIED（试图关闭风险点）；4) 补录有效 `risk_acceptance`（approver/at/evidence_id、理由/有效期）后再次推进关闭；5) 残余为 MID/LOW/VERY_LOW 时直接关闭；6) `risk_acceptance` 有效期到期 |
| 预期 | 2) 残余等级正确；3) `residual_level∈{HIGH,VERY_HIGH}` 且无有效 `risk_acceptance` 时**门控拦截、DONE→VERIFIED 被拒、风险点不可关闭**（"未接受不可关闭"）；4) 补录有效 risk_acceptance 后方可关闭、审批与签批人留痕；5) 低残余无需管理层接受即可关闭；6) 接受到期触发 REVIEW_DUE/复评；报告"已转整改/已闭环"基于 `risk_finding↔remediation_task` 关系正确呈现 |
