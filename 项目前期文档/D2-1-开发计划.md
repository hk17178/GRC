---

## 一、目的与原则
- 把"怎么干、分几期、谁干、何时交付"定下来，作为开发与里程碑管理依据。
- 原则：**风险驱动 + 分期交付 + 地基先行 + 外部依赖前置锁定**。
- 核心判断（来自可行性评估）：**拒绝 10 模块齐头并进一次交付**；优先打通纵向闭环，再做智能与外延。

---

## 二、建设范围与分期策略 ⭐

> 依据模块依赖（横向支撑 M6/M8/M9/M10 须先于业务模块）与风险（M2 表单引擎、M5/M8 隔离为最高风险）排序。

### 一期 —— 地基 + 高频闭环（MVP）
**目标：** 形成"制度 + 评估 + 整改 + 通知"可用闭环，跑通一个完整业务回路。
- M6 组织/资产台账（含 AD 同步）
- M8 统一权限与审批流（RBAC 四元组 + 组织隔离 + Flowable 审批）
- M1 制度体系管理（全生命周期 + 签署确认，知识库问答**延后到二期**）
- M2 风险评估**基础版**：表单引擎（结构配置 + 打分/报告映射两层）+ 2–3 套脚手架（等保三级 / ISO 27001 / 供应商）+ 整改工单闭环
- M9 管理看板 + 证据留痕库
- M10 统一通知中心（通道 + 规则引擎 + 分级降噪）
- 平台底座：容器化部署、Web 安装向导、一键脚本

### 二期 —— 智能 + 复用
**目标：** 引入 AI 与跨体系复用，补齐审计。
- M5 AI 智能问答（**含组织隔离越权专项测试**，安全红线）
- M1 知识库 RAG（切片 + 向量检索 + 引用溯源）
- M2 统一控件库（"具体控制点"颗粒度 + 跨体系映射 + 一次测试多体系复用）+ 更多脚手架（PBOC/PCI/27701/27018/9001）
- M3 内部审计（复用 M2 表单引擎与整改机制）

### 三期 —— 外延 + 持续监控
**目标：** 补齐法规跟踪、供应商、持续监控。
- M4 法律法规跟踪（自建爬虫 + 失效告警 + 法规-制度映射 + 变更摘要）
- M7 第三方/供应商管理（复用 M2 + 生命周期 + SLA + 事件触发复评）
- M2 KRI 持续监控（对接 SIEM/日志/漏洞，依赖外部接口就绪）

## 六、外部依赖锁定（关键前置）⭐

> 可行性评估指出：任一外部系统不配合，相关功能即落空。以下须在对应阶段**开工前**锁定接口规范与可用承诺。

| 依赖 | 涉及 | 需就绪时点 | 锁定内容 |
|---|---|---|---|

| 短信网关 | M10 一期 | 一期中 | 接口规范、调用额度 |

## 七、关键路径与依赖
- **M0 技术验证是关键路径起点**：表单引擎与组织隔离须先 PoC 通过再大规模开发。
- 一期内：M6 + M8 是其它业务模块的前置（台账与权限先行）。
- M3/M7 依赖 M2 表单引擎成熟（故置于二/三期）。
- M5、M2-KRI 依赖外部接口就绪。

---

## 八、研发过程与质量门禁
- **迭代节奏**：2 周一个 Sprint；持续集成（每次合并跑构建+测试+模块依赖检查）。
- **分支策略**：主干 + 特性分支；合并需 Code Review 通过。
- **环境**：开发 / 测试 / 预生产 / 生产（容器化一致）。
- **质量门禁**：
  - 模块边界 CI 校验（ArchUnit/Modulith，跨模块越界即失败）；
  - 单元测试覆盖关键逻辑；接口/集成测试纳入流水线；
  - **安全门禁**：依赖扫描（SCA）+ 静态扫描（SAST）；上线前安全测试/渗透 + AI 越权专项必须通过方可发布；
  - DB 迁移版本化（Flyway），评审通过方可上库。

## 九、假设与约束（需确认）
1. 并发用户数、数据量预估**待定**（需求 13.1），影响性能设计与排期，须设计阶段确认。
2. 团队规模与到位时间**待定**，直接影响并行度与总周期。
3. 外部系统接口承诺**待锁定**（§六）。
4. 监管留存期限的具体口径**待逐条核实**（需求 11.2.2）。
5. 模型部署方式（本地/内网 API）**待定**，影响 AI 首字时延指标与采购。

## 十一、风险与应对（联动风险登记册 DM-2）
| 风险 | 应对 |
|---|---|
| M2 表单引擎超期/做成半成品 | M0 先 PoC；一期只做基础版 + 2-3 脚手架；控件库放二期 |
| 数据隔离设计缺陷致后期返工 | 地基三件套已先行评审；M0 隔离 PoC |
| 外部依赖不到位 | §六 前置锁定；不就绪则相应功能延后，不阻塞主线 |
| 爬虫长期失效维护成本 | 健康监控常态化；评估采购第三方法规源替代部分自建 |
| 运营配置能力不足 | 配套管理员配置手册 + 培训；内置脚手架降低上手门槛 |
| 周期/资源不确定 | 分期交付，每期独立可用，便于按资源滚动调整 |
