## 一、目的与原则
- 把"怎么干、分几期、谁干、何时交付"定下来，作为开发与里程碑管理依据。
- 原则：**风险驱动 + 分期交付 + 地基先行 + 外部依赖前置锁定**。
- 核心判断（来自可行性评估）：**拒绝 10 模块齐头并进一次交付**；优先打通纵向闭环，再做智能与外延。

## 二、建设范围与分期策略 ⭐

> 依据模块依赖（横向支撑 M6/M8/M9/M10 须先于业务模块）与风险（M2 表单引擎、M5/M8 隔离为最高风险）排序。

### 一期 —— 地基 + 高频闭环（MVP）
**目标：** 形成"制度 + 评估 + 整改 + 通知"可用闭环，跑通一个完整业务回路。
- M6 组织/资产台账（含**多身份源适配器**：N 个 identity_source，按域路由，详见 §二·补）
- M8 统一权限与审批流（RBAC + 组织隔离 + 审批流引擎）
- M1 制度体系管理（全生命周期 + 签署确认，知识库问答**延后到二期**）
- M2 风险评估**基础版**：表单引擎（结构配置 + 打分/报告映射两层）+ 2–3 套脚手架（等保 / ISO 27001 / 供应商）+ 整改工单闭环；**风险五级口径（极低/低/中/高/极高）统一落地**
- M3 审计管理**外审闭环提前到一期**：对外回函三段漏斗（已提交外部机构→外方受理→外方确认关闭）（详见 §二·补）
- M9 管理看板 + 证据留痕库
- M10 统一通知中心（通道 + 规则引擎 + 分级降噪）
- **可配置性基线**（JSONB 扩展列 + 字段元数据登记 + 自定义视图/看板存储）与 **i18n 基础设施**（中英文切换、文案外置语言包、日期/数字/时区本地化、顶栏语言切换）
- 平台底座：容器化部署、Web 安装向导、一键脚本

### 二期 —— 智能 + 复用
**目标：** 引入 AI 与跨体系复用，补齐审计与监管事项。
- M5 AI 智能问答（**含组织隔离越权专项测试**，安全红线）
- M1 知识库 RAG（切片 + 向量检索 + 引用溯源）
- M2 统一控件库（"具体控制点"颗粒度 + 跨体系映射 + 一次测试多体系复用）+ 更多脚手架（PBOC/PCI/27701/27018/9001）
- **M3 审计管理（完整版）**：在一期外审闭环基础上补齐 **内审 + 监管检查 + 认证审计**（audit_type 区分）+ 外部发现项 + 认证有效期跟踪 + 等保测评全过程视图（复用 M2 表单引擎与整改机制）
- **M11 监管事项模块**：报送日历（法定时限预警）/ 监管问询 / 处罚约谈台账 / 重大事件报送 / 年度合规计划（IR/SOC 主体不在本平台，仅"重大事件监管报送"在本平台）
- **模型接入与管理**菜单（CR-004）：模型/端点/API Key/RAG/提示词/"数据不出域"开关（默认开）；密钥入密钥库不落明文；配置变更入操作审计；外部模型端点出站白名单防 SSRF
- **建议与反馈模块（M-FB）**（CR-004）：系统管理菜单 + AI 助手双入口收集 BUG/功能建议 → 格式化 → 分类 → AI 可行性初评（人工复核为准）→ 同步到代码仓库"反馈与建议"目录（脱敏 + 管理员审批后推送、出站审计、凭证入密钥库、可一键切内网 Git）→ M10 周期分析推送

### 三期 —— 外延 + 持续监控
**目标：** 补齐法规跟踪、供应商、持续监控。
- M4 法律法规跟踪（自建爬虫 + 失效告警 + 法规-制度映射 + 变更摘要）
- M7 第三方/供应商管理（复用 M2 + 生命周期 + SLA + 事件触发复评）
- M2 KRI 持续监控（对接 SIEM/日志/漏洞，依赖外部接口就绪）

## 二·补（V1.9 对齐 · CR-001/002/003/004）

> 本节为对齐增补，保留既有分期与编号，明确新模块、新能力的工作项与归属。产品 UI 不出现实现术语（M2/四元组/Flowable 等）。

### 1. M3 审计管理 Epic（原"内部审计"升级）
- **范围**：内审 / 外审 / 监管检查 / 认证审计，以 `audit_type` 区分；复用 M2 表单引擎与整改机制。
- **外审闭环（提前到一期）**：外审任务 + 外部发现项 + **对外回函三段漏斗**（已提交外部机构 → 外方受理 → 外方确认关闭，固化为状态机）。
- **二期补全**：内审 + 监管检查 + 认证审计；**认证有效期跟踪**（到期预警）；**等保测评全过程视图**（贯穿测评各阶段）；趋势分析。

### 2. M11 监管事项模块 Epic（新一级业务模块）
- **范围**：报送日历（法定时限预警）/ 监管问询 / 处罚约谈台账 / 重大事件报送 / 年度合规计划。
- **边界**：IR/SOC 主体不在本平台，仅"重大事件监管报送"在本平台。
- **归属**：二期（依赖 M2 表单引擎与 M10 通知）。

### 3. 多身份源适配器（CR-003，取代单域 AD 同步）
- N 个 `identity_source`（各子公司/海外独立 AD 域或林）；用户唯一键 `(domain_id, username)`。
- 登录 `DOMAIN\user` 或 `user@domain` 认证路由；各子公司管本域、集团聚合；**不依赖域间信任**；各域独立服务账号、密钥入密钥库。
- 单域故障仅影响该域并维持权限快照。
- **M0 隔离 PoC 强制覆盖**：多域同名用户、跨域 `org_id` 隔离。

### 4. 可配置性基线（CR-001-2，M0/一期）
- JSONB 扩展列 + 字段元数据登记；自定义视图/筛选/看板/报表/工作流/KPI/通知场景存储。
- 集中管控分级开放 + 发布前预览 + 变更留痕。

### 5. i18n 基础设施（M0/一期）
- 中英文切换、文案外置语言包、日期/数字/时区本地化、内容可选双语、顶栏语言切换。

### 6. CR-002 能力纳入 WBS（二期 F8）
- UAR（用户访问复核）/ SoD / 越权下钻；
- A-T-V（资产-威胁-脆弱性）/ 残余风险 / 指数下钻；
- 看板下钻 / 卷宗 / benchmark / 操作审计日志；
- 义务清单 / ROPA / 举证链；
- 供应商技术安全 DPA 闭环；
- 跨模块检索。

### 7. 模型接入与管理 + 建议与反馈（CR-004，二期）
- **模型接入与管理**（独立菜单）：模型/端点/API Key/RAG/提示词/"数据不出域"开关（默认开）；密钥入密钥库不落明文；配置变更入操作审计；外部端点出站白名单防 SSRF。
- **建议与反馈（M-FB）**：系统管理菜单 + AI 助手双入口 → 格式化 → 分类（BUG/建议）→ AI 可行性初评（人工复核为准）→ 同步到代码仓库"反馈与建议"目录（GitHub 脱敏 + 管理员审批后推送、出站审计、凭证入密钥库、可一键切内网 Git）→ M10 周期分析推送管理员/企微机器人。

### 8. 排除边界（重申）
- 漏洞闭环管理、SIEM/扫描平台本体不建设；GRC 仅接口采集 KRI。

## 七、关键路径与依赖
- **M0 技术验证是关键路径起点**：表单引擎与组织隔离须先 PoC 通过再大规模开发；隔离 PoC 须覆盖**多域同名用户**与**跨域 org_id 隔离**。
- 一期内：M6 + M8 是其它业务模块的前置（台账与权限先行）；**多身份源适配器**就绪是各子公司管本域、集团聚合的前提。
- **风险五级口径（极低/低/中/高/极高）须在 M0 统一落地**，避免 M2/M3/M9 下游返工。
- M3 完整版（内审/监管检查/认证）与 M11、M7 依赖 M2 表单引擎成熟（故置于二/三期）；**M3 外审闭环提前到一期**单独实现。
- M5、模型接入管理、M2-KRI 依赖外部接口/模型端点就绪。

## 八、研发过程与质量门禁
- **迭代节奏**：2 周一个 Sprint；持续集成（每次合并跑构建+测试+模块依赖检查）。
- **分支策略**：主干 + 特性分支；合并需 Code Review 通过。
- **环境**：开发 / 测试 / 预生产 / 生产（容器化一致）。
- **质量门禁**：
  - 模块边界 CI 校验（ArchUnit/Modulith，跨模块越界即失败）；
  - 单元测试覆盖关键逻辑；接口/集成测试纳入流水线；
  - **安全门禁**：依赖扫描（SCA）+ 静态扫描（SAST）；上线前安全测试/渗透 + AI 越权专项必须通过方可发布；
  - DB 迁移版本化（Flyway），评审通过方可上库。

## 九、假设与约束（需确认）
1. 并发用户数、数据量预估**待定**（需求 13.1），影响性能设计与排期，须设计阶段确认。
2. 团队规模与到位时间**待定**，直接影响并行度与总周期。
3. 外部系统接口承诺**待锁定**（§六）。
4. 监管留存期限的具体口径**待逐条核实**（需求 11.2.2）。
5. **AI 模型须"数据不出域"、本地/内网部署优先**（默认开启数据不出域开关）；外部模型端点须出站白名单。模型部署方式影响 AI 首字时延指标与采购。
6. **风险五级口径（极低/低/中/高/极高，VERY_LOW/LOW/MID/HIGH/VERY_HIGH）已统一**，须在 M0 落地以避免下游返工（删除一切"3 级/高中低三级"残留）。
7. 多身份源数量（N 个 identity_source）与各域服务账号/密钥就绪时点**待锁定**，影响一期适配器开发与隔离测试。
