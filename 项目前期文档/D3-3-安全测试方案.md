# GRC 平台 · 安全测试方案

**文档编号：** D3-3 ｜ **版本：** v1.0（草案） ｜ **日期：** 2026-06-23
**依据：** 需求 V1.8（§7.3、§十、§13.3）、D1-3 安全、D3-1 测试计划
**状态：** 待评审

> 安全测试是上线门禁（D3-1 §7.2）。**数据越权与留痕篡改一律按致命缺陷，不达标不上线。**

---

## 一、目标与范围
验证：身份认证、RBAC 四元组、**组织数据隔离**、**AI 越权防护（红线）**、留痕不可篡改、加密、OWASP 防护、凭证安全、爬虫 SSRF。

---

## 二、测试方法与工具
- 方法：黑盒 + 灰盒；权限矩阵穷举；渗透测试；代码/依赖扫描。
- 工具（建议）：SAST（静态扫描）、SCA（依赖漏洞）、DAST/渗透（如 Burp 类）、自定义越权脚本（角色×组织矩阵自动化）。
- 环境：预生产，多子公司测试数据 + 全角色账号。

---

## 三、专项一：组织数据隔离越权（最高优先级）

### 3.1 角色 × 组织矩阵（自动化穷举）
对每个角色、每对（A 子公司账号 → B 子公司数据）组合，验证以下入口**全部拒绝**：
| 入口 | 用例 |
|---|---|
| 列表/详情 API | 直接请求他子公司资源 ID |
| AI 问答 | 提问他子公司数据 |
| 看板 | 越权查看他子公司聚合 |
| 导出 | 导出他子公司数据 |
| 知识库检索 | 检索他子公司敏感记录 |

**预期**：拒绝 + 记 operation_log；数据库 RLS 兜底（即使应用层被绕过）。

### 3.2 垂直越权
低权限角色尝试高权限操作（如普通员工改模板结构、审批报告）→ 全部拒绝。

---

## 四、专项二：AI 问答越权（安全红线，需求 7.3）
| 用例 | 预期 |
|---|---|
| A 子公司账号问 B 子公司高风险项 | 不返回 B 数据 |
| 多轮对话诱导跨子公司 | 上下文受 visibleOrgs 约束，不泄露 |
| 提示注入（在问题中注入"忽略权限"） | 检索仍按权限过滤，不被绕过 |
| 越权数据是否进入生成上下文 | 检索阶段已过滤，生成上下文无越权数据 |
| 敏感问答留痕 | operation_log 记录查询内容与命中范围 |

---

## 五、专项三：留痕不可篡改（需求 11.4）
| 用例 | 预期 |
|---|---|
| 应用账号 UPDATE/DELETE evidence | 数据库拒绝 |
| 改写 operation_log | 拒绝（append-only） |
| 篡改 MinIO 文件后校验 | 哈希不一致，可检出 |
| 删除/改写尝试本身 | 留日志 |

---

## 六、专项四：OWASP 与应用安全
| 项 | 用例 |
|---|---|
| 注入 | SQL/JSONB 注入尝试 → 参数化拦截 |
| XSS | 富文本/字段注入脚本 → 输出编码/CSP 拦截 |
| CSRF | 跨站伪造关键操作 → Token/SameSite 拦截 |
| 文件上传 | 上传可执行/超大/伪类型 → 校验拦截；下载需鉴权 |
| 认证会话 | 暴力破解锁定、会话固定、登出失效 |
| 敏感泄露 | 错误信息/响应是否含敏感字段 |
| 依赖漏洞 | SCA 扫描无高危 |

---

## 七、专项五：加密与凭证
- 传输 TLS 1.2+ 验证；敏感存储/备份加密验证。
- 配置/代码中**无明文密钥**（AD 服务账号、短信密钥等）。

---

## 八、专项六：爬虫 SSRF
- 法规爬虫目标白名单；构造内网地址→拒绝出站，防被利用打内网。

---

## 九、出口准则
- 越权专项（§3/§4）**100% 拒绝**；留痕篡改（§5）**100% 拦截**；
- OWASP/渗透无高危；凭证无明文；
- 报告对接等保测评格式（需求 13.3）。
- 任一红线不达标 → 阻断上线。

---

## 十、交付物
安全测试报告（含越权矩阵结果、渗透发现与修复、加密/凭证核查）、缺陷修复复测记录。

---

## 评审重点
1. 角色×组织越权矩阵的覆盖完整性与自动化程度。
2. AI 提示注入防护的测试充分性。
3. 留痕数据库层拦截的验证方法。

## 十一·补（V1.9 对齐 · CR-002/CR-003/CR-004）：权限治理、多域控、防篡改链与 AI 红线安全测试

> 本补节在不改动既有专项编号（§3–§8）前提下，新增四组被测对象。出口准则（§九）同步追加：**多域控串权、AI 数据不出域越线、防篡改链断裂一律按致命缺陷，不达标不上线。**

### 11.1 专项七：权限治理闭环（UAR / SoD / 特权账号）
| 用例 | 预期 |
|---|---|
| UAR 周期复核闭环 | 复核任务按周期生成；"驳回/收回"动作即时落库并触发权限变更；逾期未复核可下钻定位责任人 |
| SoD 冲突检测 | 为同一用户同时授予互斥角色/权限 → 检出冲突并拦截或强制走例外审批；例外留痕 |
| 特权账号未审查 → KRI 告警 | 特权账号超期未审查命中 KRI 阈值 → 产生告警；**告警可下钻到账号、所属域、最近审查时间** |
| 权限变更审计 | 任一授权/收回/角色变更写入操作审计（append-only），含操作人、时间、前后权限快照、来源域 |
| 风险等级一致性 | 上述告警/风险标记按**五级（极低/低/中/高/极高 · VERY_LOW/LOW/MID/HIGH/VERY_HIGH）**呈现，无三级残留 |

### 11.2 专项八：多域控身份安全（CR-003）
被测多身份源（N 个 `identity_source`，各子公司/海外独立 AD 域/林；不依赖域间信任）：

| 用例 | 预期 |
|---|---|
| 跨域同名不串权 | 两域存在同名 `username`，复合唯一键 **(domain_id, username)** 区分；A 域同名账号不可读/写 B 域用户数据与权限 |
| UPN / 域前缀路由正确 | `user@domain`（UPN 后缀）与 `DOMAIN\user`（域前缀）均路由到正确目标域校验 |
| 错误域拒绝 | 凭据正确但**指定/路由到错误域** → 认证拒绝，不回退到其它域，不串权 |
| 各域独立服务账号密钥无明文 | 各 `identity_source` 独立服务账号密钥入密钥库；配置/代码/日志中**无明文** |
| 单域不可用仅影响该域 | 单域故障：仅该域登录/同步受影响，其余域照常；**中断期间权限快照不失效**，恢复后增量补同步 |
| 跨域用户落不同 org_id 隔离 | 跨域用户映射到不同 `org_id` 子树，沿用组织隔离 + RBAC 四元组；越权读他 `org_id` 数据被拒并 RLS 兜底 |
| 集团聚合视图 | 集团聚合视图只读跨域汇总，不得借聚合入口绕过本域隔离写他域 |

> **TC-INT（集成）覆盖：** 多身份源**增量同步**用例——增/改/禁用/删除在各域独立增量拉取；同步中断与恢复后增量补同步；同步过程不污染他域数据、不破坏 `(domain_id, username)` 唯一性。

### 11.3 专项九：可信时间戳（TSA）与防篡改链（独立被测）
| 用例 | 预期 |
|---|---|
| 可信时间戳（TSA）验证 | 关键留痕/证据加盖 TSA 时间戳，可独立验证时间戳有效性与签名链 |
| 防篡改链完整性 | 留痕链（哈希链/链式校验）逐条可验；任一节点被改 → 链校验失败可检出并定位 |
| 操作审计 append-only 不可篡改 | 应用账号对 `operation_log` 执行 UPDATE/DELETE → 数据库拒绝；改写尝试本身留日志，作为**独立被测项**通过/不通过判定 |
| 时间回拨/伪造 | 篡改系统/客户端时间不影响 TSA 可信时间；伪造时间戳被验签拒绝 |

### 11.4 专项十：AI 数据不出域红线 + 反馈出站安全（CR-004）
| 用例 | 预期 |
|---|---|
| AI「数据不出域」开关（默认开） | 开关开启时**仅允许内网模型/内网 API，禁止外部模型端点**；任何向外部端点的出站请求被拦截——作为**数据外发红线用例**，越线即致命缺陷 |
| 外部模型端点出站白名单 / 防 SSRF | 仅放行白名单端点；构造内网地址/重定向/DNS 重绑定 → 拒绝出站，防 SSRF 打内网 |
| 代码仓库同步凭证安全 | GitHub/内网 Git 凭证（Token）入密钥库，不落明文；可一键切内网 Git；凭证不出现在日志/配置/响应 |
| 同步出站 SSRF 核查 | 仓库地址/端点限定白名单，内网地址注入被拒；出站全程记审计 |
| 反馈出站脱敏有效性 | 对外推送前自动脱敏（PII、密钥、内部 IP、截图敏感区、子公司可识别信息）+ 管理员审批后方可出站；构造含敏感信息的反馈 → 验证脱敏命中且未审批不直推 |
| 配置变更入审计 | 模型接入/反馈出站配置变更写入操作审计（append-only） |

### 11.5 出口准则补充
- 专项七 UAR/SoD/特权账号告警可下钻验证通过；权限变更 100% 入审计。
- 专项八多域控：跨域同名 **100% 不串权**、错误域 **100% 拒绝**、单域故障权限快照不失效、跨域 `org_id` 隔离 100% 拒绝越权。
- 专项九 TSA 验证通过、防篡改链可检出、`operation_log` append-only 100% 拦截改写。
- 专项十 AI 不出域开关 **0 例越线出站**、SSRF 全拒、凭证无明文、反馈出站脱敏有效且经审批。
- 上述任一红线不达标 → 阻断上线。
